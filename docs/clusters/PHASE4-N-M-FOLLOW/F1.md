# Slice F1 — Chain-sync follow + op-cert counter equal-reuse

**Cluster:** PHASE4-N-M-FOLLOW
**Strengthens:** CN-PUMP-01, DC-PUMP-01, DC-PUMP-02,
DC-EVIDENCE-01, DC-EVIDENCE-02, RO-LIVE-05.
**Dependencies:** PHASE4-N-M-SCHED (committed single-block
agreed transcript stays load-bearing).

## §1 Intent

Make the admission wire pump *follow* the peer's chain in
order from the intersect point instead of admitting the
peer's tip once and idle-polling. Each `RollForward` from
chain-sync triggers a block-fetch for the rolled-forward
block's exact point. Also fix the op-cert counter rule to
permit equal counters across blocks (real pool operation).

After this slice, sustained admission against the docker
preprod peer produces N consecutive `block_admitted` events
with monotonically increasing slots, all hashes matching the
peer's chain, zero divergence.

## §2 Scope

### BLUE/GREEN code changes

- `crates/ade_runtime/src/admission/wire_pump.rs`:
  - Add `extract_chain_sync_header_point(envelope_bytes)`.
    Parses `[serialisationInfo, tag(24, bytes(header_cbor))]`,
    hashes inner with Blake2b-256 → `block_hash`, parses
    `header_body[1]` → slot, returns
    `Point::Block { slot, hash }`. Babbage/Conway Praos
    only (honest scope; Conway-only admission per N-M-C).
  - `handle_chain_sync::IntersectFound`: emit TipUpdate;
    queue chain-sync RequestNext (NOT block-fetch).
  - `handle_chain_sync::RollForward`: emit TipUpdate; call
    `extract_chain_sync_header_point` on the header bytes;
    queue block-fetch RequestRange for the extracted point.
    On extractor error → `AdmissionWirePumpError::ChainSyncDecode`
    (fail-closed). Do NOT queue chain-sync RequestNext here.
  - `handle_block_fetch::BatchDone`: already queues
    chain-sync RequestNext (unchanged). This is the
    sequencing anchor.

- `crates/ade_core/src/consensus/praos_state.rs`:
  - `OpCertCounterMap::upsert_strict`: regression rule is
    `counter < existing`. Equal is no-op. Strictly greater
    updates.

- `crates/ade_core/src/consensus/header_validate.rs`:
  - Step 4 op-cert counter check: regression rule is
    `header.op_cert_counter < existing`.

- `crates/ade_core/src/consensus/op_cert.rs::tests`:
  - Rename `apply_op_cert_rejects_equal_counter` →
    `apply_op_cert_accepts_equal_counter_as_noop` with
    inverted assertion.

- `crates/ade_core/src/consensus/praos_state.rs::tests`:
  - Rename `op_cert_upsert_rejects_equal_counter` →
    `op_cert_upsert_accepts_equal_counter_as_noop`.
  - Extend `op_cert_upsert_accepts_strictly_increasing` →
    `op_cert_upsert_accepts_monotonic_increasing` (includes
    equal-counter step).

### Test additions

In `wire_pump.rs::tests`:
- `extract_chain_sync_header_point_returns_slot_and_hash` —
  synthesizes a known-shape envelope at slot 0x1234, asserts
  the extractor returns Block { slot: 0x1234, hash: blake2b_256(inner) }.
- `extract_chain_sync_header_point_rejects_malformed_envelope`
  — exercises 4 malformed shapes (wrong outer array len,
  wrong tag number, empty input, wrong inner array len).
- `pump_emits_tip_update_and_request_next_on_intersect_found_no_block_fetch`
  — replaces the old `_and_block_on_initial_intersect_and_block_fetch`
  test: asserts that post-IntersectFound the next outbound
  is chain-sync RequestNext, NOT block-fetch.
- `rollforward_drives_block_fetch_then_request_next` — full
  flow: IntersectFound → TipUpdate → RequestNext; server
  RollForward → TipUpdate → block-fetch RequestRange (with
  the extracted slot matching the synthesized header).

## §4 Hard prohibitions

- No silent skip on malformed RollForward header bytes.
- No re-introduction of the IntersectFound tip-jump.
- No new pub fn that does header-point extraction outside
  `wire_pump::extract_chain_sync_header_point`.
- No I/O / eprintln in BLUE.
- No relaxation of the op-cert counter rule to allow
  strictly-less counters.

## §5 Non-goals

- Catching up to the live tip (operator-time, not slice
  scope).
- ChainDb persistence of admitted blocks (future slice).
- Rollback replay against already-admitted blocks.
- Block production.
- Pre-Babbage TPraos header support.
