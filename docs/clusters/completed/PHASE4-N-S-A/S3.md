**Cluster:** PHASE4-N-S-A.

real KES signature, not placeholder); DC-CONS-18 (body-hash

**Introduces:** `ProducerShell::kes_sign_header(period,

**Dependencies:** PHASE4-N-S-A A2 (HEAD `db442f2`).

## §1 New RED API

```rust
pub fn kes_sign_header(
    &self,
    period: u32,
    preimage: &UnsignedHeaderPreImage,
) -> Result<KesSignature, ShellSignError>;

pub fn kes_period_in_window(&self, period: u32) -> bool;
```

`kes_sign_header` accepts only the branded type — arbitrary-
byte signing is mechanically unrepresentable. Internally
forwards inner bytes to `kes_sign_at`.

`kes_period_in_window` is the pre-flight check the two-pass
forge uses to fail fast with `KesPeriodMismatch` BEFORE the
placeholder-signature first pass.

## §2 Two-pass forge (in `run_real_forge`)

1. **Pre-flight:** `shell.kes_period_in_window(kes_period)`.
   Fail closed → `ForgeFailed { KesPeriodMismatch }`.
2. **Pass 1 (placeholder signature):** `assemble_tick` +
   `forge_block` → `forged_placeholder` whose
   `body_hash` + `body_size` + header_body fields are
   canonical.
3. **Extract canonical fields:** `decode_block` + re-decode
   of inner block (mirrors A2's byte-match test shape).
4. **Build `vrf_result` CBOR** (mirrors forge.rs:282-288).
5. **Call BLUE recipe:**
   `unsigned_header_pre_image(...)` → `UnsignedHeaderPreImage`.
6. **RED step 3 (real):** `shell.kes_sign_header(period, &preimage)`
   → real `KesSignature`.
7. **Pass 2 (real signature):** `assemble_tick` +
   `forge_block` → `forged_real`.
8. **BLUE step 6:** `self_accept(forged_real.bytes, ...)`.

## §3 Honest scope

The N-R-A A4 integration test `full_stake_answer_reaches_self_accept_and_rejects`
still emits `ForgeFailed` (not `ForgeSucceeded`) — because
the synthetic `LedgerState` / `chain_dep_state` /
`era_schedule` / `pool_distr_view` in the test fixture
don't constitute a valid Praos chain context for OTHER
validation steps beyond KES (e.g., issuer pool lookup,
chain-dep evolution). The KES bridge IS in place; A2's
byte-match test proves the recipe is correct. Real-chain-
state fixtures are N-S-C's operator-pass deliverable.
