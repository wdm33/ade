violate specific spec rules, and prove `block_validity` rejects each

## 4. Intent

Prove no-false-accept on real-block-derived adversarial input — positive
agreement (B1-S6) is necessary but not sufficient. The dangerous failure is a
malformed/malicious block accepted as Valid; this slice makes that impossible
to do silently, by construction.

## 9. Design Summary

Each mutation takes a real corpus block, applies a single targeted corruption,
and the harness asserts `block_validity(...).verdict` is `Invalid` with the
expected `BlockRejectClass`. The **primary assertion is always**: the verdict is
NOT `Valid`. The class assertion is secondary (if a mutation lands a different
but still-fail-closed class, assert the actual class and document why).

Mutation set (each derived from a real corpus block):

| # | Mutation | Spec rule violated | Expected class |
|---|---|---|---|
| M1 | Truncate the header VRF proof (80→79 bytes) | fixed-size VRF proof | HeaderInvalid (MalformedField via `expect_size`) |
| M2 | Tamper the header VRF vkey (so `blake2b256(vkey) ≠ registered keyhash`) | VRF key binding | HeaderInvalid (VrfKeyhashMismatch) |
| M3 | Flip a byte in the KES signature | KES authenticity | HeaderInvalid (KesInvalid) |
| M4 | Set the header slot far beyond the EraSchedule horizon | forecast horizon | HeaderInvalid (HFC / OutsideForecastRange) |
| M5 | Flip a byte in the block body (header `body_hash` unchanged) | header↔body binding | BodyHashMismatch |
| M6 | Forge a witness: wrong-size / fabricated vkey+sig, **with the header `body_hash` patched** to the mutated body so it passes M5's gate and reaches body validation | Ed25519 witness verification (fail-closed) | BodyInvalid |

> M6 is the highest-value case — it proves the body authority rejects a forged
> spend fail-closed (the audit's signature-bypass class), not merely that the
> body-hash binding catches tampering. Recompute the body hash (B1-S4's helper)
> for the mutated body and patch the header so the mutation reaches step 5.

If ANY mutation yields `Valid`, STOP — it is a fail-open bug; fix the authority
(BLUE) and add the fix to this slice, do not weaken the assertion.
