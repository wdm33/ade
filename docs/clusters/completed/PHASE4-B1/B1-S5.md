## 4. Intent

Make Conway (Praos) header validation **complete and fail-closed**: verify the
single combined VRF (leader value + nonce contribution both derived from one
output), verify the KES signature + op-cert, bind the registered VRF keyhash,
and reject every malformed header field. Without this, no Conway block can be
validated and the no-false-accept guarantee (CE-B1-4) has a header-shaped hole.

## 9. Design Summary

### Proof obligation — exact Praos VRF construction

Pin the following against the Praos spec / `cardano-base` /
`ouroboros-consensus-protocol` (`Ouroboros.Consensus.Protocol.Praos.VRF`) and
**validate against the 14 real Conway-576 blocks**:

- **VRF input** (Praos `mkInputVRF`): a single input derived from `(slot, eta0)`
  — NO nonce/leader role tag (that split is TPraos). Best-known construction:
  `inputVRF = blake2b256( slot_be8 ‖ eta0_32 )`. Verify the header's single VRF
  proof under the pool VRF vkey over `inputVRF`, producing the 64-byte output.
- **Leader value**: derived from the single output (Praos `vrfLeaderValue`),
  compared to the threshold via the existing integer `is_leader`/`check_leader_claim`
  (taylorExpCmp). Pin whether the leader value is the raw output, a domain-tagged
  hash, or a `bytesToNatural` of a sub-slice — against the spec + corpus.
- **Nonce contribution**: derived from the same output (Praos `vrfNonceValue`),
  fed to `nonce::apply_nonce_input` instead of a separate nonce-role proof.
- **VRF keyhash binding**: `blake2b_256(header.vrf_vkey) == ledger_view.pool_vrf_keyhash`.

**Truth test**: for all 14 corpus blocks, the leader check passes (they are real
elected blocks) and the derived nonce contribution is deterministic. If any of
the 14 fails the leader check, the construction is wrong — fix it, do not relax.

### HeaderInput VRF representation

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum HeaderVrf {
    Tpraos { nonce_proof: VrfProof, leader_proof: VrfProof },   // Shelley..Alonzo
    Praos  { proof: VrfProof, output: VrfOutput },              // Babbage, Conway
}
// HeaderInput.{vrf_nonce_proof, vrf_leader_proof} replaced by `vrf: HeaderVrf`.
```

Update the TPraos S-B7 path + tests to the `Tpraos` variant (no behavior change).

### KES + op-cert (closed wiring)

```rust
// kes_check.rs — verify the KES signature over the header body and the op-cert
// cold-key signature, via ade_crypto::kes. Fail-closed: wrong sizes -> FieldError;
// verify==false -> HeaderValidationError::KesInvalid / OpCertInvalid (add closed
// variants to HeaderValidationError).
```

Extend `HeaderValidationError` (closed) with `KesInvalid`, `OpCertInvalid`,
`MalformedField(FieldError)`, `VrfKeyhashMismatch { expected, actual }`.

### Fail-closed field checks

A small `expect_size(kind: FieldKind, actual, expected) -> Result<(), FieldError>`
helper; every fixed-size header crypto field (vrf vkey 32, vrf proof 80, kes sig,
kes vkey 32, op-cert sig 64) goes through it — wrong size ⇒ reject, never skip.

## 13. Failure Modes

| Failure | Shape | Fail-fast? |
|---|---|---|
| Wrong-size vrf/kes/opcert field | `HeaderValidationError::MalformedField(FieldError)` | yes |
| VRF proof invalid | `HeaderValidationError::VrfCert(VerificationFailed)` | yes |
| Leader value above threshold | `HeaderValidationError::VrfCert(LeaderValueAboveThreshold)` | yes |
| VRF keyhash mismatch | `HeaderValidationError::VrfKeyhashMismatch` | yes |
| KES sig invalid | `HeaderValidationError::KesInvalid` | yes |
| Op-cert invalid | `HeaderValidationError::OpCertInvalid` | yes |
