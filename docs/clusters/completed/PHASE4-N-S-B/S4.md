## §5 Honest deferral

The cluster doc §4 CE-B-5 lists a "loopback integration test
(synthetic dialer ↔ Ade listener; reply bytes byte-identical
to the snapshot's admitted bytes)" as an exit criterion.
That test is **deferred to N-S-C C1** because it requires:

- A synthetic dialer side that handshakes + sends typed
  ChainSync / BlockFetch frames — N-S-B does not currently
  ship this; the existing `n2n_dialer` is for upstream
  consumption, not synthetic-server testing.
- A produce_mode integration that constructs
  `peer_outbound: Some(new_per_peer_outbound())` — N-S-C
  C1 will do this as part of the operator-pass binary
  wiring.

The mechanical correctness of the wire IS backed by:

- Type-level: `OutboundCommand` carries typed ServerReply
  only; no Vec<u8> tunneling.
- Existing tests: B2's outbound_command tests + B3's
  listener_loopback_handshake test (unchanged behavior with
  `peer_outbound: None`).
- CI gate: no direct transport.outbound writes from
  produce_mode.
- Structural: MuxPump's tokio::select! arm processes
  OutboundCommand via the existing `session::core::step`
  path, which is exercised by every existing inbound test.
