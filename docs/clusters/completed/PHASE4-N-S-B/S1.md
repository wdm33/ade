## §1 OQ-S-B audit resolved

`MuxPump::run()` (in
`crates/ade_runtime/src/network/mux_pump.rs:49`) is
currently a flat `loop { transport.inbound.recv().await }`.
B2's refactor converts to `tokio::select!`:

```rust
loop {
    tokio::select! {
        chunk = self.transport.inbound.recv() => { /* existing */ }
        cmd = async {
            match self.outbound_relay.as_mut() {
                Some(rx) => rx.recv().await,
                None => std::future::pending().await,
            }
        } => { /* new — encode via session, send via transport.outbound */ }
    }
}
```

**No double-mut-borrow concern:** both arms call
`mpsc::Sender::send(&self)` on `transport.outbound`, which
takes `&self`. The session-aware encoder lives inside the
new arm; it produces bytes for `transport.outbound` via the
same path `SessionEffect::SendBytes` already uses.

**`None` outbound_relay degrades cleanly:**
`std::future::pending()` never resolves, so dialer mode
(which constructs `MuxPump` with `outbound_relay: None`)
sees the select arm as inert. Existing dialer tests
continue to pass without modification.
