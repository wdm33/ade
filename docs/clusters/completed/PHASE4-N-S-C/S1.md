## §1 Operator procedure

```bash
# 1. Bring up a single-pool private testnet
# Choose either the cardano-node devnet recipe or the
# existing local-devnet harness. The operator controls all
# stake: cold/VRF/KES keys live in the operator's filesystem
# and are NEVER committed.

mkdir -p ~/.cardano-private-testnet-n-s-c1
cd ~/.cardano-private-testnet-n-s-c1

cardano-cli node key-gen \
    --cold-verification-key-file cold.vkey \
    --cold-signing-key-file cold.skey \
    --operational-certificate-issue-counter-file cold.counter
cardano-cli node key-gen-VRF \
    --verification-key-file vrf.vkey \
    --signing-key-file vrf.skey
cardano-cli node key-gen-KES \
    --verification-key-file kes.vkey \
    --signing-key-file kes.skey
cardano-cli node issue-op-cert \
    --kes-verification-key-file kes.vkey \
    --cold-signing-key-file cold.skey \
    --operational-certificate-issue-counter-file cold.counter \
    --kes-period 0 \
    --out-file node.opcert

# 2. Bring up cardano-node in private-testnet mode
# (pseudo-config; consult ouroboros-network testnet docs).
docker run -d --name cardano-node-c1-peer \
    --network host \
    -v $(pwd):/keys:ro \
    ghcr.io/intersectmbo/cardano-node:11.0.1 \
    cardano-node run \
      --genesis-file /keys/shelley-genesis.json \
      --conway-genesis /keys/conway-genesis.json \
      --shelley-vrf-key /keys/vrf.skey \
      --shelley-kes-key /keys/kes.skey \
      --shelley-operational-certificate /keys/node.opcert \
      --port 3010

# 3. Run ade_node in produce mode against the Haskell peer
ade_node --mode produce \
    --listen 127.0.0.1:3001 \
    --cold-skey cold.skey \
    --kes-skey kes.skey \
    --vrf-skey vrf.skey \
    --opcert node.opcert \
    --genesis-file shelley-genesis.json \
    --evidence-log evidence.jsonl

# 4. Wait for a slot at which the operator's pool is leader
# (eligibility under operator-controlled stake). The
# `LeaderCheckOutcome` event in evidence.jsonl will show
# is_leader=true at that slot. The `BlockForged` event
# follows.

# 5. Capture peer acceptance
docker logs cardano-node-c1-peer 2>&1 > peer.log

# 6. Filter to the matching block hash
HASH=$(jq -r 'select(.kind=="BlockForged").block_hash' evidence.jsonl | head -1)
grep -F "$HASH" peer.log | grep -E "BlockAccepted|AddedToCurrentChain"
```

## §3 Honest scope

C1 PROVES the engineering bridge end-to-end: Ade can forge
a block, serve it via N2N block-fetch, and cardano-node
accepts it on a controlled private testnet.

C1 does NOT prove the bounty-facing claim because the
testnet is operator-controlled (not the public preprod
chain). C2 captures the bounty-facing evidence against
docker `cardano-node-preprod`.
