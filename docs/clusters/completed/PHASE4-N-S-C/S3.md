## §3 What N-S accomplished

The producer-mode block-production acceptance leg is
**mechanically backed end-to-end** through the bridge:

```
ProducerShell signing keys (RED)
  ↓
[BLUE leader-check] verify_and_evaluate_leader
  ↓ Eligible
[BLUE recipe] unsigned_header_pre_image → UnsignedHeaderPreImage
  ↓ branded type
[RED] shell.kes_sign_header(period, &preimage) → real KesSignature
  ↓
[GREEN] assemble_tick → ProducerTick with real signature
  ↓
[BLUE] forge_block → ForgedBlock
  ↓
[BLUE] self_accept → AcceptedBlock
  ↓
[N-R-B] BroadcastBlock → ServedChainSnapshot::push_atomic
  ↓
[N-R-B] dispatch_server_frame_event_to_outbound
  ↓ typed ServerReply (no Vec<u8> tunnel)
[N-S-B] OutboundCommand via per-peer mpsc::Sender
  ↓
[N-S-B] MuxPump::handle_outbound_command
  ↓ session::core::step (existing session-aware encoder)
[N-S-B] MuxPump::route_effect (SendBytes) → TCP socket
  ↓
Peer
```

Every arrow has a type-level + test-level proof. The
remaining gap is operator action: bringing up a private
testnet OR provisioning preprod stake + running the binary
+ capturing the paired log.
