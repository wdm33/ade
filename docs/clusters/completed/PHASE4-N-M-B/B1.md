# Slice B1 — AgreementVerdict + AdmitOutcome + verdict::derive

**Cluster:** PHASE4-N-M-B
**CEs:** CE-N-M-B-6 (DC-ADMIT-01/06), CE-N-M-B-4 (DC-ADMIT-08)
**Dependencies:** none
**Registry effects on merge:** DC-ADMIT-01 + DC-ADMIT-06 +
DC-ADMIT-08 → `enforced`.

## Intent

Pure GREEN evidence reducer comparing CN-CONS-08's admit
verdict + peer's announced tip → closed `AgreementVerdict`. No
authority, no I/O, no clock. Per
[[feedback-evidence-reducers-are-green-not-authority]].

## Scope

- `crates/ade_node/src/admission/verdict.rs` (new GREEN):
  - `pub enum AgreementVerdict { Agreed{..}, Lagging{..}, Diverged{..}, InputNotFound{..} }` — 4 variants, narrow semantics per cluster doc §1 (I-B1).
  - `pub enum AdmitOutcome { Valid{ slot, block_hash, post_fp }, Invalid{ slot, block_hash, reason }, InputMissing{ tx_in } }` — closed input sum derived from BLUE `BlockValidityVerdict`.
  - `pub fn derive(outcome: &AdmitOutcome, peer_tip: &Tip) -> AgreementVerdict` — pure reducer.
- `crates/ade_node/src/admission/mod.rs` — module barrel.
- `ci/ci_check_lagging_is_evidence_only.sh` — forbids `Lagging`
  matched in success-result patterns outside the verdict reducer
  + its tests.

## Reducer semantics

| Input                                                      | Output                                  |
|------------------------------------------------------------|-----------------------------------------|
| `AdmitOutcome::Valid { slot, block_hash, .. }`, peer tip slot == slot, peer hash == block_hash | `Agreed { our_hash, peer_hash }` |
| `AdmitOutcome::Valid { slot, block_hash, .. }`, peer tip slot > slot                            | `Lagging { our_slot, peer_slot }` |
| `AdmitOutcome::Valid { slot, block_hash, .. }`, peer tip slot == slot, peer hash != block_hash | `Diverged { our_hash, peer_hash, slot }` |
| `AdmitOutcome::Invalid { slot, .. }`, peer announced block at slot                              | `Diverged { our_hash, peer_hash, slot }` |
| `AdmitOutcome::InputMissing { tx_in }`                     | `InputNotFound { tx_in }` |
| peer tip is `Origin`                                       | `Lagging` (we're always at-or-ahead of Origin) |

## §14 Hard Prohibitions

- No I/O / clock / state / unsafe / panic in `verdict.rs`.
- No `HashMap` / `HashSet` / float.
- No second `pub fn derive` returning `AgreementVerdict`.
- No "Healthy" / "Ready" / "Synced" variant on
  `AgreementVerdict`.
- No code path (outside reducer body + tests) that maps
  `Lagging` to a success sentinel.

## §15 Non-Goals

- No JSONL writer (B2).
- No admission tokio runner (B4).
- No actual call to `admit_via_block_validity` here — that's
  B4's job. B1 builds the closed types + reducer the runner
  composes over.
