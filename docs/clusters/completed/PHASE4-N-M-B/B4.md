# Slice B4 — admission tokio runner + per-admit WAL append + verdict emit

**Cluster:** PHASE4-N-M-B
**CEs:** CE-N-M-B-1 (CN-ADMIT-01), CE-N-M-B-6 (DC-ADMIT-02/03/05)
**Dependencies:** B1, B2, B3 + N-M-A + N-L + N-K stacks
**Registry effects on merge:** CN-ADMIT-01 + DC-ADMIT-02 + DC-ADMIT-03 + DC-ADMIT-05 → `enforced`.

## Intent

Single tokio admission runner composing the full stack:
- import JSON seed (N-M-A);
- mint BootstrapAnchor;
- seed_to_snapshot → persistent snapshot;
- bootstrap_initial_state warm-start (CN-NODE-01);
- spawn N2nDialer per peer (N-L);
- per AdmittedBlock orchestrator effect: WalStore::append +
  verdict::derive + AdmissionLogEvent::AgreementVerdict;
- Diverged | InputNotFound → AdmissionHalted, exit fatal.
- SIGINT/SIGTERM → drain, AdmissionShutdown, exit 0.

## Scope

- `crates/ade_node/src/admission/runner.rs` (new RED):
  - `pub async fn run_admission(inputs: AdmissionInputs) -> AdmissionExitCode` — sole entry point.
  - `pub struct AdmissionInputs` — closed bundle of cli + writer + wal_store + shutdown signal.
  - Closed `AdmissionExitCode` mirroring the binary exit code constants.
- Exit code constants:
  - `EXIT_LIVE_AGREEMENT_DIVERGED = 30`
  - `EXIT_LIVE_INPUT_NOT_FOUND = 31`
- `ci/ci_check_admission_runner_closure.sh` — single
  `pub fn run_admission` + single `pub fn seed_to_snapshot`
  across the workspace.

## Runner flow

```text
emit AdmissionStarted { peer_count, json_seed_path, wal_dir }

let (utxo, utxo_fp) = import_cardano_cli_json_utxo(&cli.json_seed)?;
let initial_ledger_fp = seed_to_snapshot(utxo, chain_dep_seed, seed_point, &mut snapshot_store)?;
emit SnapshotImported { seed_point_slot, imported_utxo_fp_hex, utxo_entry_count }

let anchor = mint(MintInputs { ... });
let (ledger, chain_dep, tip) = bootstrap_initial_state(...)?;
emit BootstrapComplete { initial_ledger_fp_hex, chain_tip_slot }

verify wal_store.verify_chain(&anchor.initial_ledger_fingerprint)?

spawn N2nDialer per peer
loop on orchestrator inbox:
  match event:
    OrchestratorEffect::AdmittedBlock { slot, hash } =>
      let admit_outcome = build_admit_outcome(...);
      let post_fp = fingerprint(&state.receive_state.ledger).combined;
      let prev_fp = wal_store.read_all().last().map(|e| e.post_fp()).unwrap_or(anchor.initial_ledger_fingerprint);
      wal_store.append(WalEntry::AdmitBlock { prior_fp: prev_fp, block_hash: hash, slot, verdict: BlockVerdictTag::Valid, post_fp })?;
      emit BlockAdmitted { slot, block_hash_hex, post_fp_hex }
      let verdict = verdict::derive(&admit_outcome, &peer_tip);
      emit AgreementVerdict { kind, ... per variant }
      match verdict:
        Diverged{..} | InputNotFound{..} =>
          emit AdmissionHalted { reason: ... }
          return fatal exit code
        Agreed | Lagging => continue

on SIGINT/SIGTERM:
  drain pipeline
  emit AdmissionShutdown { reason: SignalReceived }
  exit 0
```

## §12 Mechanical Acceptance Criteria

- [ ] `run_admission_emits_start_then_snapshot_then_bootstrap_then_loops`.
- [ ] `run_admission_appends_one_wal_entry_per_admitted_block`.
- [ ] `run_admission_emits_exactly_one_agreement_verdict_per_admit`.
- [ ] `run_admission_halts_fatal_on_diverged` (exit code 30).
- [ ] `run_admission_halts_fatal_on_input_not_found` (exit code 31).
- [ ] `run_admission_emits_admission_shutdown_on_signal`.
- [ ] `ci/ci_check_admission_runner_closure.sh` passes.

## §14 Hard Prohibitions

- No second `pub fn run_admission`.
- No bypass of `bootstrap_initial_state`.
- No silent retry after Diverged / InputNotFound.
- No mixing wire-only literals in the runner module.
- No skip of WAL append on a successful admit.
- No `unwrap()` / `panic!()` in the runner body — all errors
  map to closed `AdmissionExitCode` variants.

## §15 Non-Goals

- No new event variants (B2 defines the closed set).
- No new verdict variants (B1 defines the closed set).
- No real live cardano-node peer — the integration test in B6
  uses a hermetic loopback responder.
