# Slice A1 — JSON UTxO seed importer

**Cluster:** PHASE4-N-M-A
**CEs:** CE-N-M-A-1 (CN-SEED-01), CE-N-M-A-2 (DC-SEED-01)
**Dependencies:** none
**Registry effects on merge:** CN-SEED-01 + DC-SEED-01 → `enforced`.

## Intent

Convert a cardano-cli `query utxo --whole-utxo --out-file utxo.json`
output into a canonical Ade `(UTxOState, UtxoFingerprint)` pair.
Single authority; bit-identical across runs; no second importer
in the workspace.

## Scope

- **BLUE** `crates/ade_codec/src/shelley/utxo_seed_json.rs`:
  - Pure JSON parser via `serde_json`.
  - `JsonSeedError` closed sum (`Io`, `Json`, `BadAddress`,
    `BadTxInKey`, `UnknownNetwork`).
  - `pub fn parse_utxo_seed_json(bytes: &[u8]) -> Result<Vec<(TxIn, TxOut)>, JsonSeedError>`.
- **GREEN** `crates/ade_runtime/src/seed_import.rs`:
  - `pub fn import_cardano_cli_json_utxo(path: &Path) -> Result<(UTxOState, UtxoFingerprint), JsonSeedError>`.
  - Reads file, calls BLUE parser, inserts pairs into `BTreeMap`,
    computes `UtxoFingerprint` (Blake2b-256 over canonical CBOR
    `map(N) [TxIn → TxOut]` in BTreeMap iteration).
- New canonical type: `UtxoFingerprint(pub Hash32)` in
  `ade_ledger::utxo` (newtype for `Hash32`, derive
  `Debug, Clone, Copy, PartialEq, Eq, Hash`).

## JSON shape (cardano-cli 11.0.1 output)

```json
{
  "abc...#0": {
    "address": "addr_test1...",
    "datum": null,
    "datumhash": null,
    "inlineDatum": null,
    "referenceScript": null,
    "value": {
      "lovelace": 1000000,
      "policy_id_hex": { "asset_name_hex": 5 }
    }
  },
  "...": { ... }
}
```

- Key: `"<tx_hash_hex>#<output_ix>"`.
- Value: per-TxOut structured record.
- We re-encode the structured record via existing
  `ade_codec::shelley::tx_components::encode_tx_out` (composing
  the existing canonical TxOut encoder over JSON-sourced fields).

## §12 Mechanical Acceptance Criteria

- [ ] `utxo_seed_parses_minimal_two_entry_fixture` —
  hand-crafted 2-entry JSON → `Vec<(TxIn, TxOut)>` with right
  shape.
- [ ] `utxo_seed_two_imports_byte_identical` — same JSON →
  same `UtxoFingerprint` across two imports.
- [ ] `utxo_seed_btree_order_independent_of_json_order` — same
  JSON keys in shuffled order → same fingerprint.
- [ ] `utxo_seed_rejects_unparseable_json` →
  `JsonSeedError::Json`.
- [ ] `utxo_seed_rejects_bad_txin_key` (`txhash` not 64 hex
  chars, ix not numeric) → `JsonSeedError::BadTxInKey`.
- [ ] `utxo_seed_rejects_bad_address` (not bech32) →
  `JsonSeedError::BadAddress`.
- [ ] `ci/ci_check_seed_import_closure.sh` — single pub
  `import_cardano_cli_json_utxo` in the workspace.

## §14 Hard Prohibitions

- No `HashMap` / `HashSet` in `utxo_seed_json.rs` (use BTreeMap
  or Vec).
- No wall-clock / rand / float.
- No second `pub fn parse_utxo_seed_json` /
  `pub fn import_cardano_cli_json_utxo` anywhere.
- No silent partial seed on parse error — first error in any
  entry fails the whole import.

## §15 Non-Goals

- No CSV / YAML / Mithril importer (separate clusters).
- No live cardano-cli RPC; consume static JSON file only.
- No incremental import; one-shot per call.
