# Slice S3 — Expanded-skey + signature serde + period inference

**Cluster:** PHASE4-N-P
**Strengthens:** I3 (round-trip identity), I5 (period uniquely
inferable from tree shape), D5 (deterministic codec), N2 (wrong
size → fail-closed), N3 (malformed sub-tree → closed error), N4
(period overflow → fail-closed), N5 (re-serialize identity), N6
(no leakage in errors).
**Introduces:** `KesParseError` enum;
`raw_serialize_signing_key_kes` / `raw_deserialize_signing_key_kes`
methods on every Sum_n; `raw_serialize_signature_kes` /
`raw_deserialize_signature_kes` methods; the module-level
`period_from_zeroed_sum6_tree_shape` function matching the S1
proof obligation.
**Dependencies:** S1 (proof obligation doc committed at HEAD
`c04dc0b`) + S2 (BLUE algorithm committed at HEAD `80966f6`).

## §1 Intent

Implement the spec from
[`period-from-zeroed-sum6-tree-shape-proof.md`](period-from-zeroed-sum6-tree-shape-proof.md)
and ship the round-trip + cross-impl tests it mandates. After S3
the on-disk format is fully exercised; S4 layers real cardano-cli
ground-truth on top.

Also migrate `ade_crypto::kes::verify_kes_signature` from
`cardano-crypto`'s `Sum6Kes::verify_kes` to ours. After S3 there
are no production callers of `cardano_crypto::kes::Sum6Kes` in
`ade_crypto`; the import path lives only under `#[cfg(test)]`.
(The `ade_runtime::producer::signing` migration is still S5.)

## §2 Scope

### Per-impl on every `KesAlgorithm`

- `const LEVEL: u32` — `Sum0Kes::LEVEL = 0`,
  `SumKes<D>::LEVEL = D::LEVEL + 1`. Used by recursion to surface
  the level in `KesParseError` variants.
- `fn raw_serialize_signing_key_kes(sk: &Self::SigningKey) -> Vec<u8>` —
  recursive: `serialize(sk_child) || seed_or_zero (32) || vk0 (32)
  || vk1 (32)` at Sum levels; the 32-byte Ed25519 seed at Sum0.
- `fn raw_deserialize_signing_key_kes(bytes: &[u8]) -> Result<Self::SigningKey, KesParseError>` —
  size-checked at entry (`WrongPayloadSize` if mismatch); per-level
  vk-consistency walk per the proof doc §7; recursive into child.
- `fn raw_serialize_signature_kes(sig: &Self::Signature) -> Vec<u8>` —
  recursive: `serialize(sigma) || vk0 (32) || vk1 (32)` at Sum
  levels; 64-byte Ed25519 sig at Sum0.
- `fn raw_deserialize_signature_kes(bytes: &[u8]) -> Result<Self::Signature, KesParseError>` —
  size-checked entry; recursive.
- `fn current_period_of_signing_key(sk: &Self::SigningKey) -> u32` —
  recursive: walks the tree, accumulates `2^(level-1)` at each
  level where `r1_seed.is_none()`. For Sum6, returns 0..=63 by
  construction.

### `KesParseError`

`crates/ade_crypto/src/kes_sum/errors.rs` (new):

```rust
#[derive(Debug, PartialEq, Eq)]
pub enum KesParseError {
    WrongPayloadSize { actual: usize },
    LeafSignKeyAllZero,
    InconsistentSubtreeVkLeft { level: u32 },
    InconsistentSubtreeVkRight { level: u32 },
    LevelOutOfRange { level: u32 },
    InvalidEd25519SignatureLength { actual: usize },
}
```

Per the S1 proof doc §5: variants carry only `level: u32` and
`actual: usize`. No raw key bytes; no hex; no decimal seed runs.
The `Debug` derive is fine because every variant payload is a
non-secret primitive.

### `ade_crypto::kes::verify_kes_signature` migration

Currently calls
`cardano_crypto::kes::Sum6Kes::raw_deserialize_signature_kes` +
`Sum6Kes::verify_kes`. Migrate to:

```rust
let sig_inner = Sum6Kes::raw_deserialize_signature_kes(sig.0.as_ref())
    .map_err(|_| CryptoError::MalformedSignature { ... })?;
Sum6Kes::verify_kes(&vk.0, period.0, msg, &sig_inner)
    .map_err(|_| ...)
```

After this, `ade_crypto/src/kes.rs` has **no production
`cardano_crypto::kes::Sum6Kes` imports**. The
`ci_check_kes_sum_compatibility.sh` gate (S4) will mechanically
enforce this.

## §4 Hard prohibitions

- **N3.** No best-effort guesswork. Every malformed-tree
  scenario maps to a specific `KesParseError` variant.
- **N5.** Re-serialize identity: for every parsed skey,
  `serialize(parse(serialize(sk))) == serialize(sk)` byte-for-byte.
- **N6.** `KesParseError` variants carry only `level: u32` and
  `actual: usize`; never raw bytes. The fail-closed `Debug` is
  the derive, since every payload is non-secret.
- No `unsafe`, no `transmute`, no `pub(crate)` access pattern
  attempting to construct upstream `SumSigningKey` from raw
  bytes (N9).

## §5 Replay obligations preserved

- T-DET-01 + T-ENC-01 strengthened. The canonical serialization
  produces deterministic bytes for every (seed, period) pair.
