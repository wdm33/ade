## 9. Design Summary

Two mutation families, each → `tx_validity` verdict asserted **Invalid** (primary
assertion: NOT Valid; class assertion secondary):

**(A) Witness mutations on REAL corpus txs (track_utxo=false)** — for corpus txs
that carry tx-derived required signers (explicit/withdrawal/cert/voter):
| # | Mutation | Expected class |
|---|---|---|
| W1 | Remove a required tx-derived witness | MissingRequiredSigner |
| W2 | Flip a byte in a required witness signature | WitnessInvalid (InvalidWitnessSignature) |
| W3 | Truncate a witness sig/vkey | MalformedField |
| W4 | Replace a witness with one signing a different body | WitnessInvalid |

**(B) Synthetic adversarial txs (track_utxo=true, controlled UTxO)** — full
control, covers the UTxO-dependent surface that track_utxo=false defers:
| # | Mutation | Expected class |
|---|---|---|
| S1 | Value imbalance (outputs + fee ≠ inputs) | Phase1Invalid |
| S2 | Missing input-payment-key witness | MissingRequiredSigner |
| S3 | Unresolvable / dangling input | Phase1Invalid |
| S4 | Forged input-payment witness (wrong sig) | WitnessInvalid |

The harness must select real corpus txs that actually have tx-derived
requirements for family (A); if a corpus tx has none, removing a witness is not a
required-signer violation at track_utxo=false (correctly), so don't assert
Invalid for those — pick txs with withdrawals/certs/voting/explicit-signers.

If ANY mutation yields Valid → STOP, it's a fail-open bug; fix the authority
(BLUE) and document it. Never soften the assertion.
