# PHASE4-N-T · S2 — GREEN `ChainEvolution` typestate module (standalone)

> Parent: [`cluster.md`](./cluster.md). Addresses **CE-T-5, CE-T-6, CE-T-6b, CE-T-7** and the **DC-PROD-03** anchor.
> Standalone: this slice ships a new, fully-unit-tested GREEN module. It is **not wired**
> into produce_mode (that is S3). Mergeable + correct on its own (new module + tests; no
> existing behavior changes).

## Goal

A pure, linear typestate that threads the producer's chain state forward across forges:
seed from the bootstrap triple, hand out the forge-context inputs it owns, and `advance`
on a forged block by re-deriving the post-state and reconstructing the `AcceptedBlock`
token **only** through the BLUE authorities — never minting it.

## New module: `crates/ade_runtime/src/producer/chain_evolution.rs` (GREEN-by-content)

Carry the same `// Core Contract:` banner as `coordinator.rs`. Pure: no `tokio`, no I/O,
no clock, no `HashMap`/`HashSet`, no float. `BTreeMap` is fine. Add
`pub mod chain_evolution;` to `crates/ade_runtime/src/producer/mod.rs`.

### Grounding (verified — do not re-derive)
- `ade_ledger::block_validity::transition::block_validity(&LedgerState, &PraosChainDepState, &EraSchedule, &dyn LedgerView, &[u8]) -> BlockValidityOutcome { verdict: BlockValidityVerdict, ledger: LedgerState, chain_dep: PraosChainDepState }`. On `Valid`, `outcome.ledger`/`outcome.chain_dep` are the **post-state** (see `bootstrap.rs:280`).
- `ade_ledger::producer::self_accept::self_accept(&[u8], &LedgerState, &PraosChainDepState, &EraSchedule, &dyn LedgerView) -> Result<AcceptedBlock, SelfAcceptError>`. It calls `block_validity` internally and **returns only the token** (discards post-state). `AcceptedBlock` has a private constructor — it can be obtained ONLY from `self_accept`.
- `ade_ledger::consensus_view::PoolDistrView` is a concrete value (`#[derive(Clone, PartialEq, Eq)]`) that `impl LedgerView`. Hold it by value and use `&self.pool_distr_view` as the `&dyn LedgerView`. (No held trait object — OI-T.2 satisfied.)
- Decode a block's `(slot, hash)` via `ade_ledger::block_validity::decode_block(&[u8]) -> Result<_, _>` → `.header_input.slot`, `.block_hash` (see `served_chain_handle.rs:102` / `self_accept.rs` tests).
- `ChainTip` to thread: reuse `ade_runtime::producer::coordinator::ChainTip { slot: u64, block_hash: [u8;32], block_number: u64 }` (already the coordinator's tip shape) OR a local minimal `{ slot, hash, block_number }`. Prefer reusing the coordinator `ChainTip` to avoid a parallel type. Confirm Clone/PartialEq.

### Types
```rust
pub struct ChainEvolution {
    // fixed for the run (single-epoch cold-start scope)
    era_schedule: EraSchedule,
    pool_distr_view: PoolDistrView,   // doubles as the &dyn LedgerView
    eta0: Nonce,
    // evolving
    base_ledger: LedgerState,
    base_chain_dep: PraosChainDepState,
    tip: Option<ChainTip>,            // None at cold-start seed; Some after first advance
}

#[derive(Debug, Clone, PartialEq)]
pub enum ChainEvolutionError {
    /// self_accept rejected the forged bytes (validator said Invalid).
    SelfAcceptRejected(ade_ledger::producer::self_accept::SelfAcceptError),
    /// block_validity (Valid?) and self_accept (Ok?) disagreed on identical
    /// inputs — defense-in-depth against an arg-mismatch bug in advance.
    /// Structurally unreachable when advance passes identical args
    /// (self_accept calls block_validity); the guard makes that contract explicit.
    AuthorityMismatch,
}
```

### Methods
- `seed(base_ledger, base_chain_dep, tip: Option<ChainTip>, era_schedule, pool_distr_view, eta0) -> ChainEvolution`.
- Accessors (borrowing): `base_ledger(&self) -> &LedgerState`, `base_chain_dep(&self) -> &PraosChainDepState`, `era_schedule(&self) -> &EraSchedule`, `pool_distr_view(&self) -> &PoolDistrView`, `eta0(&self) -> &Nonce`, `tip(&self) -> Option<&ChainTip>`, `next_block_number(&self) -> u64` (= tip.block_number+1, or 0 if None), `prev_hash(&self) -> Hash32` (= tip.block_hash, or `Hash32([0;32])` if None).
- `advance(self, forged_bytes: &[u8]) -> Result<(ChainEvolution, AcceptedBlock), ChainEvolutionError>`:
  ```rust
  let outcome = block_validity(&self.base_ledger, &self.base_chain_dep,
                               &self.era_schedule, &self.pool_distr_view, forged_bytes);
  let bv_valid = matches!(outcome.verdict, BlockValidityVerdict::Valid { .. });
  let sa = self_accept(forged_bytes, &self.base_ledger, &self.base_chain_dep,
                       &self.era_schedule, &self.pool_distr_view);
  reconcile_verdicts(bv_valid, sa.is_ok())?;                 // -> AuthorityMismatch on disagreement
  let token = sa.map_err(ChainEvolutionError::SelfAcceptRejected)?;  // handles the Invalid path
  // bv_valid is true here; outcome.ledger/chain_dep are the post-state.
  let decoded = decode_block(forged_bytes) /* infallible here: self_accept already decoded */;
  let new_tip = ChainTip { slot: decoded.header_input.slot.0,
                           block_hash: decoded.block_hash.0,
                           block_number: self.next_block_number() };
  Ok((ChainEvolution { base_ledger: outcome.ledger, base_chain_dep: outcome.chain_dep,
                        tip: Some(new_tip), ..self_fixed_fields.. }, token))
  ```
  `advance` consumes `self` (linear typestate — a stale base cannot be reused).
- `fn reconcile_verdicts(block_validity_valid: bool, self_accept_ok: bool) -> Result<(), ChainEvolutionError>`: returns `Err(AuthorityMismatch)` iff `block_validity_valid != self_accept_ok`, else `Ok(())`. Small pure helper — the mechanical home of CE-T-6b.

### Tests (`#[cfg(test)]` in the module; use `ade_testkit::validity::ConwayValidityCorpus` like `self_accept.rs`/`bootstrap.rs` tests — `ade_runtime` already has the `ade_testkit` dev-dep, see `bootstrap.rs` tests)
- `advance_threads_post_state_forward` (CE-T-6): seed from a corpus-derived `(ledger_at_576, chain_dep eta0=corpus.epoch_nonce, pool_distr_view from corpus, era_schedule, eta0)`, `tip=None`; `advance(lightest_corpus_block)` → `Ok((evo2, token))`; assert `evo2.tip()` is `Some` with `block_number == 0`, the post-ledger/post-chain_dep differ from the seed (or at least `evo2.base_chain_dep() != seed.base_chain_dep()` — nonce evolves), and `token.as_bytes() == block`.
- `advance_two_runs_byte_identical` (CE-T-6, DC-PROD-03): seed two identical ChainEvolutions, `advance` the same block on both, assert `fingerprint(evo_a.base_ledger()).combined == fingerprint(evo_b.base_ledger()).combined` and `evo_a.base_chain_dep() == evo_b.base_chain_dep()` and `evo_a.tip() == evo_b.tip()`.
- `advance_rejects_invalid_bytes` (CE-T-6): flip a body byte (mirror `self_accept.rs::flip_body_byte`) → `advance` returns `Err(ChainEvolutionError::SelfAcceptRejected(_))`; original `self` is consumed (no partial advance).
- `reconcile_verdicts_*` (CE-T-6b): `reconcile_verdicts(true, true) == Ok`; `reconcile_verdicts(false, false) == Ok`; `reconcile_verdicts(true, false) == Err(AuthorityMismatch)`; `reconcile_verdicts(false, true) == Err(AuthorityMismatch)`.
