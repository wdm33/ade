## 1. What an Invariant Slice Is
A mergeable unit that strengthens exactly one invariant family and leaves the system fully
correct and replay-equivalent on merge. This slice delivers the Conway withdrawals decoder and
exact sum; it adds no conservation-equation wiring.

## 4. Intent
Make it impossible to feed the conservation equation a withdrawals total that is anything other
than the exact sum of a fully, canonically decoded withdrawals map: a Conway withdrawals field
is either fully decoded into a deterministic `BTreeMap<RewardAccount, Coin>` and summed exactly,
or it is a structured decode reject. A partial, truncated, duplicate-key, or overflowed
withdrawal map is never summed as if complete.

## 9. Design Summary
- `RewardAccount([u8; 29])` is a fixed-width newtype; the decoder rejects any key whose
  bytestring is not exactly 29 bytes (`CodecError::InvalidLength`).
- `decode_withdrawals` reads a **definite-length** CBOR map (indefinite → reject), each entry
  `bytes => uint`. Wrong key/value major type → `CodecError::UnexpectedCborType`. A coin value
  exceeding `u64::MAX` → reject (no truncation). **Duplicate keys → `CodecError::DuplicateMapKey`**
  (never last-wins). Trailing bytes after the map → `CodecError::TrailingBytes`.
- The result is a `BTreeMap` keyed by `RewardAccount`, giving deterministic iteration regardless
  of input key order. Strict canonical key *ordering* is NOT enforced here (the tx body hash, via
  preserved bytes, already binds the wire order; `BTreeMap` normalizes for summation). Duplicate
  *keys* are still rejected because they make the map ambiguous.
- `withdrawals_sum` accumulates in `i128` (each value ≤ `u64::MAX`, count bounded by tx size;
  total cannot exceed `i128`), returning the exact consumed-side term. No float, no rounding.

## 13. Failure Modes
All deterministic, fail-fast, structured `CodecError`; never a partial sum:
- **Truncated / `UnexpectedEof`** — input ends mid-map.
- **Indefinite-length map** — `InvalidCborStructure` (closed grammar requires definite).
- **Key not bytes / value not uint** — `UnexpectedCborType`.
- **Reward account ≠ 29 bytes** — `InvalidLength`.
- **Coin value > `u64::MAX`** — `InvalidLength`/`UnexpectedCborType` (no truncation).
- **Duplicate key** — `DuplicateMapKey`.
- **Trailing bytes** — `TrailingBytes`.

Replay impact: deterministic; same input → same reject class.

## 14. Hard Prohibitions

### Inherited Cluster-Level Prohibitions
All "Forbidden During This Cluster" items from `docs/clusters/PHASE4-B3/cluster.md` — notably:
no silent skip; fail-closed on malformed input; `HashMap`/float forbidden in BLUE; no re-encode
for any hash.

### Slice-Specific Prohibitions
- Summing a partially decoded map (any malformed entry rejects the whole field).
- Last-wins on duplicate keys (must reject).
- Truncating or saturating a coin value that exceeds `u64::MAX` (must reject).
- Using `HashMap` for the withdrawals map (must be `BTreeMap` for deterministic iteration).
- `String`/`anyhow`/formatted errors in the BLUE decode path.

**If any of these appear, the slice is incorrect.**
