### Real-oracle resolution (DONE)
- **79 inputs** resolved (spend + collateral + reference) for the **29** cert/withdrawal-bearing
  corpus txs — committed `corpus/validity/conway_epoch576/{resolution_set.txt,resolved_inputs.json}`.
  Method: cloned the 300 GB ImmutableDB EBS, `db-truncater --truncate-after-slot 163900638` (tip =
  block 12270961, the block immediately before the first corpus block, seeded by the S3
  `snapshot_163468813` boundary ledger snapshot + ~431K-slot replay), then `cardano-cli query utxo`
  against the frozen node (no-peer topology) → 44 inputs from the pre-window ledger; the remaining
  35 are **intra-corpus** (created by earlier corpus txs, resolved from corpus tx outputs via
  `resolve_b3_intra_corpus`). Recipe + verification: `corpus/tools/extract_conway_resolved_inputs.md`.
- Corpus certs are all **Neutral** (StakeDelegation/VoteDelegation) → no registration/pool/DRep
  state needed, only input `(coin,address)`.
- **Result:** `cargo test -p ade_testkit --test conway_conservation_positive_corpus` PASS — the
  **10 non-Plutus** cert/withdrawal txs → Valid at `track_utxo=true`; verdict stream byte-identical
  across two runs. The 19 Plutus-script-bearing txs (`script_data_hash` present) are carved out
  (phase-2 aiken Conway divergence, CE-88, externally blocked — orthogonal to conservation).
