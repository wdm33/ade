correct and replay-equivalent on merge. This slice is the deposit-param authority foundation
of PHASE4-B3; it adds no accounting logic.

## 4. Intent
Make it impossible for Conway value-conservation accounting to read a deposit/refund amount
from any source other than canonical ledger state — by giving the two Conway-only deposit
params (`drep_deposit`, `gov_action_deposit`) a canonical, era-present typed home, including
them in the replay fingerprint, and threading the full deposit-param set into the validator.
Begins enforcement of `DC-TXV-07`.

---

## 5. Scope

- **Modules / crates:**
  - `ade_ledger::pparams` — new `ConwayOnlyDepositParams { drep_deposit, gov_action_deposit }`
    and the validator-boundary view `ConwayDepositParams { key_deposit, pool_deposit,
    drep_deposit, gov_action_deposit }`; constructor/accessors. (`key_deposit`/`pool_deposit`
    remain on the shared `ProtocolParameters` — Shelley+, already canonical.)
  - `ade_ledger::state` — `LedgerState` gains
    `conway_deposit_params: Option<ConwayOnlyDepositParams>` (`Some` iff era == Conway;
    `None` otherwise — **absent, not defaulted**).
  - `ade_ledger::fingerprint` — `fingerprint_pparams` extended to fold
    `drep_deposit`/`gov_action_deposit` into the state fingerprint **only when
    `conway_deposit_params` is `Some`**; versioned migration.
  - `ade_ledger::conway` — `validate_conway_state_backed` signature gains a
    `ConwayDepositParams` view assembled at call sites from canonical state; forwarded to
    `check_conway_coin_conservation` (signature extended; value consumed in B3-S4).
  - `ade_ledger::rules`, `ade_ledger::tx_validity::phase1` — call sites assemble the view
    from `state.protocol_params` + `state.conway_deposit_params` and pass it in.
  - `ade_testkit::harness::snapshot_loader` (RED) — populate
    `LedgerState.conway_deposit_params` from the `ConwayGovParams` it already parses; stop
    being the value's read-path for BLUE.
  - `ci/ci_check_deposit_param_authority.sh` — NEW enforcement script.
- **State machines affected:** none (no new transition; `LedgerState` shape extended).
- **Persistence impact:** state fingerprint shape change for Conway states (replay-critical) —
  versioned migration; non-Conway fingerprints byte-unchanged.
- **Network-visible impact:** none.

**Out of scope:** deposit/refund accounting logic and the early-out removal (B3-S4); cert and
withdrawals decoders (B3-S2/S3); any `EraProtocolParameters` enum migration of fee/size/reward
params (explicitly not done — the 101 generic read sites stay on the existing struct).

## 9. Design Summary
- `drep_deposit`/`gov_action_deposit` become a typed `ConwayOnlyDepositParams` carried as
  `LedgerState.conway_deposit_params: Option<ConwayOnlyDepositParams>` — `Some` only for
  Conway, so no Conway-only field is representable (let alone defaulted) in a non-Conway state.
- The validator boundary receives a `ConwayDepositParams` *view* assembled from the two
  canonical sources (`protocol_params.{key_deposit,pool_deposit}` +
  `conway_deposit_params.{drep_deposit,gov_action_deposit}`) — single source of truth per
  field, no duplication.
- Replay completeness: the deposit params affect authoritative verdicts, so they enter the
  state fingerprint. The migration is versioned and Conway-gated so non-Conway replay goldens
  are provably untouched; Conway goldens are regenerated intentionally.
- Authority is mechanically gated by `ci_check_deposit_param_authority.sh`: no BLUE path may
  read deposit values from `ConwayGovParams`, defaults, or constants.

## 10. Changes Introduced

### Types
- **New:** `ConwayOnlyDepositParams { drep_deposit: Coin, gov_action_deposit: Coin }`
  (narrow canonical state type); `ConwayDepositParams { key_deposit, pool_deposit,
  drep_deposit, gov_action_deposit }` (validator-boundary view).
- **New error:** `ValidationEnvironmentError::MissingConwayDepositParams` — a
  validation-*environment* error, structurally distinct from tx-validity (so "bad validator
  environment" is never confused with "bad transaction").
- **Modified:** `LedgerState` gains `conway_deposit_params: Option<ConwayOnlyDepositParams>`.
  `validate_conway_state_backed` and `check_conway_coin_conservation` signatures gain
  `ConwayDepositParams` (forwarded; consumed in B3-S4).

### State Transitions
- None added. `LedgerState` shape extended; `Default` sets `conway_deposit_params: None`.

### Persistence
- `fingerprint_pparams` extended: when `conway_deposit_params` is `Some`, fold
  `drep_deposit`/`gov_action_deposit` into the canonical buffer under a versioned tag;
  non-Conway path unchanged. Conway-state fingerprint goldens regenerated.

### Removal / Refactors
- BLUE call sites stop reading deposit values from anywhere but canonical state. Testkit
  `ConwayGovParams` becomes a snapshot-decode intermediate that *writes*
  `conway_deposit_params`, never a BLUE read-path.

## 13. Failure Modes
- **Conway state missing deposit params (`conway_deposit_params == None` where era == Conway):**
  deterministic fail-fast `ValidationEnvironmentError::MissingConwayDepositParams` at view
  assembly — a validation-environment error, **not** a tx-validity error; never substitute a
  default. Affects replay: fail-fast halts deterministically.
- **Snapshot loader cannot parse `drep_deposit`/`gov_action_deposit`:** RED loader returns a
  structured load error; no partial/defaulted canonical state is written.
- **Fingerprint migration applied to non-Conway state:** must be impossible (Conway-gated);
  guarded by `pparams_fingerprint_stable_for_non_conway`.

## 14. Hard Prohibitions

### Inherited Cluster-Level Prohibitions
All "Forbidden During This Cluster" items from `docs/clusters/PHASE4-B3/cluster.md` — notably:
sourcing any deposit param from testkit/default/shell/ambient `ConwayGovParams` in BLUE;
defaulting a Conway-only param into a non-Conway semantic path; silent fingerprint change
(migration must be explicit); `HashMap`/float/wall-clock in BLUE; re-encoding for any hash.

### Slice-Specific Prohibitions
- Representing `drep_deposit`/`gov_action_deposit` as defaulted fields on a non-Conway state
  (must be `None`, structurally absent).
- Duplicating `key_deposit`/`pool_deposit` into `ConwayOnlyDepositParams` (single source of
  truth on the shared struct).
- Migrating fee/size/reward params to any `EraProtocolParameters` enum (out of scope; would
  touch 101 unrelated read sites and all-era determinism).
- Reading deposit values back from the testkit `ConwayGovParams` object in any BLUE path.
- Conflating `MissingConwayDepositParams` with a tx-validity reject class.
- Shipping the Conway fingerprint change without regenerating + reviewing the affected goldens.

**If any of these appear, the slice is incorrect.**

## 15. Explicit Non-Goals
This slice MUST NOT:
- Implement deposit/refund/withdrawal accounting or remove the conservation early-out (B3-S4).
- Add the Conway cert or withdrawals decoders (B3-S2/S3).
- Introduce an `EraProtocolParameters` enum for fee/size/reward params.
- Introduce new protocol versions or feature flags.
- Optimize performance.
