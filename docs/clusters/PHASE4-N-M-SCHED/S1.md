# Slice S1 — Wire canonical.epoch_no into the imported-window era schedule

**Cluster:** PHASE4-N-M-SCHED
**Strengthens:** CN-CONS-IN-01, DC-VIEW-01.
**Closes:** `DC-EVIDENCE-01` + `RO-LIVE-05` `open_obligation`
(`blocked_until_consensus_inputs_eta0_extraction`).
**Dependencies:** PHASE4-N-M-FRAG (wire-pump reassembly +
adjacent fixes — without those, no block ever reaches
header_validate; the epoch-number bug was masked).

## §1 Intent

Replace `make_schedule_for_imported_window`'s hardcoded
`start_epoch: EpochNo(0)` with the imported bundle's actual
`epoch_no`. After the fix, `era_schedule.locate(slot).epoch`
equals the bundle's epoch for every slot in the
`[epoch_start_slot, epoch_end_slot)` window, which is the
gating axis `LiveLedgerView` uses for all per-pool lookups.

## §2 Scope

### BLUE/GREEN file changes

- `crates/ade_node/src/admission/bootstrap.rs`:
  - Add `EpochNo` to the imports from `ade_types`.
  - `make_schedule_for_imported_window(epoch_start_slot: &SlotNo, epoch_no: EpochNo) -> EraSchedule`
    — new second parameter; the `EraSummary` constructed
    inside uses `start_epoch: epoch_no` instead of the prior
    `EpochNo(0)`.
  - Call site: pass `canonical.epoch_no`.

### Test additions

- Unit test
  `crates/ade_node/src/admission/bootstrap.rs::tests::imported_window_schedule_uses_bundle_epoch`
  (or similar): build a schedule for `epoch_no=291,
  epoch_start_slot=124_070_400`, assert
  `schedule.locate(124_140_000).epoch == EpochNo(291)`.
