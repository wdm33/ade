# PHASE4-N-V · S1 — `ade_codec::encode_block_envelope` (new BLUE encoder)

> Parent: [`cluster.md`](./cluster.md). Addresses **CE-V-2, CE-V-3** + declares
> `CN-FORGE-03`. Standalone: a new canonical block-envelope encoder + its
> round-trip tests. Not wired into `forge_block` yet (that is S2). Mergeable on
> its own.

## Goal

A single canonical block-envelope **encoder**, inverse to the existing
`decode_block_envelope`, so the producer (S2) and validator share one envelope
grammar.

## Grounding (verified)
- `ade_codec::cbor::envelope::decode_block_envelope(&[u8]) -> Result<BlockEnvelope { era, block_start, block_end }, CodecError>` requires `[era, block]`: `array(2)`, era uint (`CardanoEra::try_from(u8)`), then the inner block item; no trailing bytes (envelope.rs:45–87).
- `CardanoEra::Conway = 7`; `CardanoEra::as_u8()` returns the discriminant (era.rs:74–87,149).
- CBOR writers available in `ade_codec::cbor`: `write_array_header(buf, ContainerEncoding::Definite(2, IntWidth::Inline))`, `write_uint_canonical(buf, n)` (used in forge.rs).
- `ade_codec` is BLUE and depends only on `ade_types` — it **cannot** use `ConwayValidityCorpus` (that lives in `ade_testkit`, which dev-deps `ade_codec`). The corpus-pin test therefore lives in `ade_ledger` (which has `ade_testkit` as a dev-dep and re-exports `decode_block_envelope`/corpus access).

## Edits

### 1. `crates/ade_codec/src/cbor/envelope.rs` (BLUE, new fn)
```rust
/// Encode the era-tagged block envelope `[era, block]` — the inverse of
/// [`decode_block_envelope`]. `block_bytes` is the era-specific inner block
/// (already canonically encoded). Produces `array(2) ‖ uint(era) ‖ block_bytes`.
pub fn encode_block_envelope(era: CardanoEra, block_bytes: &[u8]) -> Vec<u8> {
    let mut out = Vec::with_capacity(2 + block_bytes.len());
    cbor::write_array_header(&mut out, ContainerEncoding::Definite(2, IntWidth::Inline));
    cbor::write_uint_canonical(&mut out, era.as_u8() as u64);
    out.extend_from_slice(block_bytes);
    out
}
```
(Confirm the exact `write_array_header` / `write_uint_canonical` / `ContainerEncoding` / `IntWidth` paths + signatures against `forge.rs` usage. Carry the BLUE `// Core Contract:` banner conventions of the file.)

### 2. Tests in `crates/ade_codec/src/cbor/envelope.rs` (`#[cfg(test)]`)
- `encode_decode_block_envelope_round_trips` (**CE-V-2**): for several `(era, inner)` pairs (incl. `Conway` + a synthetic inner like `[0x83,0x01,0x02,0x03]`), `let e = encode_block_envelope(era, inner); let env = decode_block_envelope(&e).unwrap();` assert `env.era == era` and `&e[env.block_start..env.block_end] == inner`.
- `conway_envelope_head_is_82_07`: `encode_block_envelope(CardanoEra::Conway, &[0x80])` starts with `[0x82, 0x07]`.

### 3. Corpus-pin test in `crates/ade_ledger/` (`#[cfg(test)]`, has corpus dev-dep) (**CE-V-3**)
- `encode_block_envelope_reencodes_corpus_block_identically`: take a `ConwayValidityCorpus` block `b`; `let env = decode_block_envelope(&b).unwrap();` (era should be `Conway`); `let inner = &b[env.block_start..env.block_end];` assert `ade_codec::cbor::envelope::encode_block_envelope(env.era, inner) == b` (byte-identical re-encode of a real block — pins the `82 07` head + exact envelope shape against real data; resolves OQ1). Put it where the corpus is already used, e.g., alongside `self_accept.rs` tests or a new `crates/ade_ledger/tests/envelope_corpus_pin.rs`.
