# PHASE4-N-T · S3 — Wire `ChainEvolution` + real forge context + delete synthetic

> Parent: [`cluster.md`](./cluster.md). Addresses **CE-T-8, CE-T-9** and the
> produce-mode side of **CE-T-5/CE-T-6** (real wiring). Strengthens CN-PROD-02,
> CN-FORGE-01, DC-CONS-18 (recorded at S5 close).
> After S3 the node forges a **coherent, chain-extending** sequence from real
> bootstrap state (if it is a leader). Blocks are **not served yet** (the
> `BroadcastBlock` arm stays a no-op until S4) — that keeps S3 correct: it forges

## Goal

Replace the synthetic never-leader forge scaffolding with a real
`ChainEvolution`-derived `ForgeRequestContext` (real `LeaderScheduleAnswer` for
the operator's pool, real base state, **absolute slot**), and `advance` the
`ChainEvolution` on each `ForgeSucceeded`. Delete `SyntheticForgeInputs` +
`build_synthetic_forge_context` + the synthetic answer helpers.

## Grounding (verified — do not re-derive)
- `ade_core::consensus::leader_schedule::query_leader_schedule(&LeaderScheduleQuery { slot: SlotNo, pool: Hash28 }, ledger_view: &dyn LedgerView, era_schedule: &EraSchedule, state: &PraosChainDepState) -> Result<LeaderScheduleAnswer, LeaderScheduleError>` (leader_schedule.rs:74). It uses `state.epoch_nonce` for the VRF-input `alpha` and reads `pool_vrf_keyhash`/`pool_active_stake`/`total_active_stake`/`active_slots_coeff` from the ledger_view; `UnknownPool` if the operator's pool is absent or total stake is 0.
- Operator pool id: `Hash28(ade_crypto::blake2b_224(&shell.cold_vk().0).0)` (mirrors `producer_shell.rs:174`). `ProducerShell` exposes `cold_vk() -> Ed25519VerificationKey`, `vrf_verification_key()`, `public_metadata()`, `opcert()`.
- `run_real_forge(slot, kes_period, &ForgeRequestContext, &mut ProducerShell) -> CoordinatorEvent` (produce_mode.rs:582) is complete (incl. the N-S-A real KES-signs-unsigned-header bridge). It uses `ctx.eta0` for the VRF input — so **`ctx.eta0` MUST be the same nonce `query_leader_schedule` used** (`state.epoch_nonce`). Set `ctx.eta0 = &chain_evo.base_chain_dep().epoch_nonce` and pass `chain_evo.base_chain_dep()` to `query_leader_schedule` — single source.
- `ChainEvolution` (S2): `seed(...)`, accessors `base_ledger/base_chain_dep/era_schedule/pool_distr_view/eta0/tip/next_block_number/prev_hash`, `advance(self, &[u8]) -> Result<(ChainEvolution, AcceptedBlock), ChainEvolutionError>` (consumes self).
- S1 already computes `boot_ledger`, `boot_chain_dep`, `boot_tip`, `real_era_schedule`, `real_pool_distr` and `consensus` (the bundle) in `run_produce_mode`.

## Edits (all in `crates/ade_node/src/produce_mode.rs`)

1. **Seed `ChainEvolution`** after the S1 bootstrap, before the loop:
   ```rust
   let mut chain_evo: Option<ChainEvolution> = Some(ChainEvolution::seed(
       boot_ledger, boot_chain_dep, boot_tip,
       real_era_schedule, real_pool_distr,
       consensus.epoch_nonce.clone(),   // eta0 == epoch_nonce
   ));
   ```
   (Hold as `Option` so the consuming `advance` can `take()`/replace inside the loop.)
2. **Absolute start slot** (CE-T-9): initialize the slot counter from the bootstrap tip, not `0`:
   ```rust
   let mut current_slot: u64 = boot_tip.as_ref().map(|t| t.slot + 1)
       .unwrap_or(consensus.epoch_start_slot.0);   // cold-start: first slot of the imported window
   ```
   (Keep the per-tick `current_slot += 1` increment + the `--max-slots` cap semantics.)
3. **Operator pool id** (once, before the loop):
   `let pool_id = ade_types::Hash28(ade_crypto::blake2b_224(&shell.cold_vk().0).0);`
   and a `let pparams = ProtocolParameters::default();` (real pparams fidelity is out of N-T scope — keep default, as the synthetic path did).
4. **Replace the synthetic forge handler.** In the `RequestForge` arm of the forge handler (`apply_effects_with_forge_handler`, produce_mode.rs:917), replace the `synthetic.leader_schedule_answer_for_slot(...)` + `as_context_with_answer_and_vk(...)` path with:
   ```rust
   let evo = chain_evo.as_ref().ok_or("chain_evo consumed")?;
   let answer = match query_leader_schedule(
       &LeaderScheduleQuery { slot: ade_types::SlotNo(*slot), pool: pool_id.clone() },
       evo.pool_distr_view(), evo.era_schedule(), evo.base_chain_dep(),
   ) {
       Ok(a) => a,
       // Unknown pool / outside horizon ⇒ we are not a leader this slot.
       Err(_) => { /* feed CoordinatorEvent::ForgeNotLeader { slot, vrf_output_fingerprint: [0;8] } to coordinator_step and continue */ }
   };
   let vrf_vk = shell.vrf_verification_key();
   let ctx = ForgeRequestContext {
       eta0: &evo.base_chain_dep().epoch_nonce,
       vrf_vk: &vrf_vk,
       leader_schedule_answer: &answer,
       pparams: &pparams,
       base_state: evo.base_ledger(),
       chain_dep_state: evo.base_chain_dep(),
       era_schedule: evo.era_schedule(),
       pool_distr_view: evo.pool_distr_view(),
       block_number: ade_types::BlockNo(evo.next_block_number()),
       prev_hash: evo.prev_hash(),
       protocol_version: ProtocolVersion { major: 9, minor: 0 },
       prev_opcert_counter: None,
   };
   let event = run_real_forge(*slot, *kes_period, &ctx, shell);
   ```
5. **Advance on `ForgeSucceeded`.** After `run_real_forge` returns, if the event is `ForgeSucceeded { artifact, .. }`, advance the chain BEFORE/with feeding the event to `coordinator_step`:
   ```rust
   if let CoordinatorEvent::ForgeSucceeded { artifact, .. } = &event {
       let evo = chain_evo.take().ok_or("chain_evo consumed")?;
       match evo.advance(&artifact.bytes) {
           Ok((next_evo, _accepted)) => { chain_evo = Some(next_evo); }   // _accepted used by S4
           Err(e) => {
               // Fail-closed: do not advance; surface as a forge failure for this slot.
               chain_evo = None; // or re-seed-free: leave None and emit ForgeFailed
               eprintln!("ade_node produce: chain advance rejected: {e:?}");
               // feed CoordinatorEvent::ForgeFailed { slot, reason: Other } instead of ForgeSucceeded
           }
       }
   }
   let (new_state, more_effects) = coordinator_step(prev_state, event)?;
   ```
   (Borrow/move care: `event` is consumed by `coordinator_step`; extract `artifact.bytes` by clone if needed for `advance`. The `_accepted` token is dropped in S3; S4 routes it to `push_atomic`.)
6. **Delete** `SyntheticForgeInputs` (struct + impl), `build_synthetic_forge_context`, `leader_schedule_answer_for_slot`, `as_context_with_answer_and_vk`, and the `synthetic_forge` local + its construction at produce_mode.rs:166. Remove the now-unused `synthetic: &SyntheticForgeInputs` param from `apply_effects_with_forge_handler` (replace with the `chain_evo` + `pool_id` + `pparams` it now needs). **(CE-T-8)**
