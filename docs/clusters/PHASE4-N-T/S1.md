# PHASE4-N-T · S1 — Real bootstrap-state startup (behavior-preserving)

> Parent: [`cluster.md`](./cluster.md). Slice spec is normative for the implementer.
> Addresses **CE-T-3, CE-T-4** (and the bootstrap-call half of CE-T-12, completed in S5).
> **Behavior-preserving:** after S1 the node still never forges (the synthetic
> zero-stake `LeaderScheduleAnswer` still gates the forge); S1 only makes the
> real bootstrap state + real `PoolDistrView` + real `EraSchedule` available and
> threads them into the forge context's *base* (S3 swaps the answer + adds
> `ChainEvolution`). Do **not** introduce `ChainEvolution` here (that is S2/S3).

## Goal

`produce_mode` builds the real `(LedgerState, PraosChainDepState, Option<ChainTip>)`
from the operator seed and routes it through `bootstrap_initial_state`'s
**cold-start** branch, instead of fabricating ledger state. Adds the
`PoolDistrView` projection + `EraSchedule` from the consensus-inputs bundle.

## Grounding (verified facts — do not re-derive)

- `ade_runtime::bootstrap::bootstrap_initial_state(BootstrapInputs { chaindb, snapshot_store, era_schedule, ledger_view, genesis_initial }) -> Result<(LedgerState, PraosChainDepState, Option<ChainTip>), BootstrapError>` (bootstrap.rs:88). **Cold-start branch** (bootstrap.rs:102) fires when `chaindb.tip()` is `None` AND `snapshot_store.list_snapshot_slots()` is empty, and returns `genesis_initial` unchanged with `tip = None`. It does **not** touch `ledger_view` on cold-start.
- `ade_runtime::chaindb::InMemoryChainDb` implements both `ChainDb` + `SnapshotStore`; a fresh `InMemoryChainDb::new()` is empty ⇒ forces cold-start.
- `import_cardano_cli_json_utxo(&Path) -> Result<(UtxoState, Hash32 /*fp*/), _>` (`ade_runtime::seed_import`). Used by admission/bootstrap.rs:119.
- `import_live_consensus_inputs(&Path) -> Result<LiveConsensusInputsCanonical, LiveConsensusInputsImportError>` (`ade_runtime::consensus_inputs`). Canonical fields: `epoch_no: EpochNo`, `epoch_start_slot: SlotNo`, `active_slots_coeff: ActiveSlotsCoeff`, `epoch_nonce: Nonce`, `pool_distribution: BTreeMap<Hash28, consensus_inputs::PoolEntry { active_stake: u64, .. }>`, `pool_vrf_keyhashes: BTreeMap<Hash28, Hash32>`.
- Ledger seed (admission pattern, bootstrap.rs:175): `let mut ledger = LedgerState::new(CardanoEra::Conway); ledger.utxo_state = utxo;` and `chain_dep = PraosChainDepState::genesis(canonical.epoch_nonce.clone())`.
- **OI-T.3 RESOLVED:** `ade_ledger::consensus_view::PoolDistrView` *is* a `LedgerView` (consensus_view.rs:`impl LedgerView for PoolDistrView`) and is already used as the `ledger_view` arg to `block_validity` in `bootstrap.rs` tests. So build **one** `PoolDistrView` and use it as both the forge `pool_distr_view` and the `&dyn LedgerView`. No `LiveLedgerView` in produce_mode.

### 2. `crates/ade_node/src/produce_mode.rs`
- Add a GREEN-pure projection helper (no I/O, no clock):
  ```rust
  /// Project the operator consensus-inputs bundle into the leadership
  /// PoolDistrView. total_active_stake = sum of pool active stakes;
  /// per-pool vrf_keyhash from the bundle's pool_vrf_keyhashes map.
  /// PoolDistrView impls LedgerView, so it doubles as the &dyn LedgerView
  /// for block_validity / self_accept (OI-T.3).
  fn pool_distr_view_from_consensus_inputs(
      c: &LiveConsensusInputsCanonical,
  ) -> PoolDistrView {
      use ade_ledger::consensus_view::PoolEntry;
      let mut pools: BTreeMap<Hash28, PoolEntry> = BTreeMap::new();
      let mut total: u64 = 0;
      for (pool, entry) in &c.pool_distribution {
          total = total.saturating_add(entry.active_stake);
          let vrf_keyhash = c.pool_vrf_keyhashes.get(pool).copied()
              .unwrap_or(Hash32([0u8; 32]));
          pools.insert(pool.clone(), PoolEntry { active_stake: entry.active_stake, vrf_keyhash });
      }
      PoolDistrView::new(c.epoch_no, total, c.active_slots_coeff, pools)
  }
  ```
