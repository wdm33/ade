## §2 OQ8 fix (CN-SNAPSHOT-02 enforcement)

`crates/ade_network/src/block_fetch/server.rs::producer_block_fetch_serve`
now performs an endpoint-presence check after
`served.range_bytes(from, to)` returns:

```rust
let first_key = entries.first().map(|(s, h, _)| (*s, h.clone())).expect(...);
let last_key = entries.last().map(|(s, h, _)| (*s, h.clone())).expect(...);
if first_key != from || last_key != to {
    return Ok((state, BlockFetchServerStep::Replies(vec![ServerReply::no_blocks()])));
}
```

The previous implementation returned "whatever was in the
BTreeMap range" — a permissive ad-hoc partial response that
diverged from the canonical cardano-node behavior. The fix
matches Haskell `ouroboros-network`'s typed-protocols
reference: both endpoints + every block between MUST be
present, else `NoBlocks`.

3 new tests in
`crates/ade_network/src/block_fetch/server.rs::tests`:

- `n_r_b_partial_overlap_from_endpoint_not_in_snapshot_yields_no_blocks`
- `n_r_b_partial_overlap_to_endpoint_not_in_snapshot_yields_no_blocks`
- `n_r_b_partial_overlap_both_endpoints_fabricated_yields_no_blocks`

All 3 pass; existing `producer_block_fetch_serve` tests
continue to pass.
