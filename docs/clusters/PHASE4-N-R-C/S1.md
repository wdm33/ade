# Slice C1 — Opcert envelope parser

**Cluster:** PHASE4-N-R-C.
**Strengthens:** `CN-PROD-02` (opcert parser sub-item).
**Introduces:** `ade_runtime::producer::opcert_envelope` +
declares + flips `CN-OPCERT-01` and `DC-OPCERT-01` to
`enforced`.

## §1 Intent

Land the real opcert envelope parser. Consumes the OQ4
fixtures captured in N-R-A A1 (a real `cardano-cli node
issue-op-cert` output against cardano-node 11.0.1 +
cardano-cli 11.0.0.0).

## §2 Closed pipeline

1. JSON-decode + extract `type`, `cborHex`.
2. Envelope-type check: must equal
   `NodeOperationalCertificate`.
3. Hex-decode `cborHex` (lowercase + uppercase accepted).
4. CBOR `array(2)` header.
5. Element 0 — `array(4)` of `[bytes(32), uint, uint,
   bytes(64)]`.
6. Element 1 — `bytes(32)` cold VK.
7. Construct `OperationalCert` from element 0.

## §3 Closed error surface

```rust
pub enum OpCertParseError {
    JsonShape,
    WrongEnvelopeType,
    MalformedCborHex,
    MalformedCbor,
    HotVkeyWrongLength { found: usize },
    SigmaWrongLength { found: usize },
    ColdVkWrongLength { found: usize },
}
```

No `String` payloads in load-bearing variants.
