**2 documented bridges deferred to future post-N-R clusters
(NOT a bounty win):**

1. **KES-signs-real-unsigned-header bridge.** Currently
   `run_real_forge` step 3 KES-signs a placeholder
   (`expected_vrf_input` bytes). The real Praos protocol
   requires KES to sign the CBOR-encoded unsigned header
   body. Without this, `self_accept` correctly rejects the
   forged block (KES signature won't verify) → no
   `ForgeSucceeded` ever reaches step 6. Tracked in
   `CN-FORGE-01.open_obligation` and
   `CN-SNAPSHOT-01.open_obligation`.

2. **MuxPump outbound-relay extension.** Currently
   `dispatch_server_frame_event` computes response bytes
   but cannot transmit them back to the peer (MuxPump owns
   the outbound channel; produce_mode has no path to send
   into it). Without this, peers cannot fetch forged blocks
   even if forging worked. Tracked in
   `CN-PROD-01.open_obligation`.
