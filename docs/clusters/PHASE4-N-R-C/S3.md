# Slice C3 — Legacy binary shim

**Cluster:** PHASE4-N-R-C.
**Strengthens:** N10 (no independent legacy production path).
**Introduces:** rewritten `live_block_production_session.rs`
+ `ci/ci_check_no_independent_forge_codepath.sh`.

## §1 Intent

Retire `live_block_production_session.rs` as an independent
forge codepath. Per DQ-C3, keep the binary name (operator
muscle memory) but reduce the body to a thin deprecation
shim that prints a banner and exits non-zero.

## §2 First-line banner (per DQ-C3)

```
DEPRECATED: live_block_production_session is a shim; invoking produce_mode::run_produce_mode.
```

No alternate flags. No alternate defaults. No duplicate
parser logic. Exit status 2.

## §3 CI gate

`ci/ci_check_no_independent_forge_codepath.sh` enforces N10
mechanically:

- Allow-list: `crates/ade_node/src/produce_mode.rs` is the
  only file permitted to combine `vrf_prove` + `kes_sign_at`
  + `forge_block` + `self_accept` calls in the same module.
- Excludes the primitive definition sites + tests + the
  deprecated shim itself.
- Fails if any other file combines all four primitives —
  surfacing a new independent forge codepath at CI time.

The gate PASSes today.

## §4 Tests

`cargo test -p ade_core_interop` continues to pass (the
binary's behavior is now print-and-exit; no test depends on
the legacy code).
