## 9. Design Summary

### Trait refinement (honest correction)

The S-B6 `LedgerView` exposed `pool_vrf_key(epoch, pool) -> Option<VrfVerificationKey>`.
The ledger does NOT hold the VRF *vkey* — it holds the registered VRF *keyhash*
(`blake2b-256(vrf_vkey)`); the actual vkey arrives in the block header. Replace:

```rust
// ade_core::consensus::ledger_view
pub trait LedgerView {
    fn total_active_stake(&self, epoch: EpochNo) -> Option<u64>;
    fn pool_active_stake(&self, epoch: EpochNo, pool: &Hash28) -> Option<u64>;
    // CHANGED: the ledger holds the registered VRF keyhash, not the vkey.
    fn pool_vrf_keyhash(&self, epoch: EpochNo, pool: &Hash28) -> Option<Hash32>;
    fn active_slots_coeff(&self, epoch: EpochNo) -> Option<ActiveSlotsCoeff>;
}
```

Header validation (B1-S5) does the binding: `blake2b_256(header.vrf_vkey) ==
pool_vrf_keyhash`. Update `leader_schedule` (it only looked up the key to confirm
the pool is known — switch to `pool_vrf_keyhash`), the testkit stub, and the
`leader_schedule_corpus` test to the new method. No behavior regression.

### Production projection

```rust
// ade_ledger::consensus_view
use ade_core::consensus::ledger_view::LedgerView;
use ade_core::consensus::vrf_cert::ActiveSlotsCoeff;
use ade_types::{EpochNo, Hash28, Hash32};
use std::collections::BTreeMap;

/// The leadership-relevant projection (mirrors a LedgerState `nesPd` /
/// pool-distribution). For B1 it is built from the B1-S1 corpus; later it can
/// be built from a parsed Conway LedgerState. Pure data; no I/O.
pub struct PoolDistrView {
    epoch: EpochNo,
    total_active_stake: u64,
    asc: ActiveSlotsCoeff,
    pools: BTreeMap<Hash28, PoolEntry>,   // pool_id -> entry
}
struct PoolEntry { active_stake: u64, vrf_keyhash: Hash32 }

impl PoolDistrView {
    pub fn new(epoch: EpochNo, total_active_stake: u64, asc: ActiveSlotsCoeff,
               pools: BTreeMap<Hash28, PoolEntry>) -> Self;
}
impl LedgerView for PoolDistrView { /* total/pool stake, vrf_keyhash, asc;
    None for unknown epoch or pool */ }
```

The B1-S1 corpus stores `sigma = active_stake / total` per pool with a shared
denominator (`= pdTotalActiveStake`). So `total_active_stake = denominator`,
`pool_active_stake = numerator`. A GREEN helper in `ade_testkit::validity`
builds a `PoolDistrView` from `ConwayValidityCorpus` for the tests.
