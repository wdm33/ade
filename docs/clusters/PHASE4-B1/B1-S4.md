## 9. Design Summary

```rust
// ade_ledger::block_validity::transition
pub struct BlockValidityOutcome {
    pub verdict: BlockValidityVerdict,
    pub ledger: LedgerState,            // evolved iff Valid, else == input clone
    pub chain_dep: PraosChainDepState,  // evolved iff Valid, else == input clone
}

pub fn block_validity(
    ledger:       &LedgerState,
    chain_dep:    &PraosChainDepState,
    era_schedule: &EraSchedule,
    ledger_view:  &dyn LedgerView,
    block_cbor:   &[u8],
) -> BlockValidityOutcome;
```

### Algorithm (fail-fast, total)

1. **Decode** `block_cbor` (era-tagged) via `ade_codec` → era, header, body bytes.
   Decode failure → `Invalid { class: BodyInvalid, error: Body(LedgerError::Decoding..) }`,
   states unchanged.
2. **Build `HeaderInput`** from the decoded header (Conway → `HeaderVrf::Praos`,
   carrying the cert's output+proof; pre-Conway → `Tpraos`). Map op-cert / KES.
3. **Header authority**: `validate_and_apply_header(chain_dep, header_input,
   ledger_view, era_schedule)`. Error → `Invalid { class: HeaderInvalid,
   error: Header(e) }`, states unchanged (FAIL-FAST: body NOT run).
4. **Body-hash binding** (wired, not assumed): compute the era-correct block
   body hash and assert it equals the validated header's `body_hash`. Use the
   existing helper if `ade_ledger`/`ade_codec` provides one; otherwise compute
   it (era-correct: blake2b-256 over the body segments). Mismatch →
   `Invalid { class: BodyHashMismatch, error: BodyHashMismatch{header, actual} }`,
   states unchanged. (This is the E5-class "wire the check" obligation — prove
   it with a negative test that an altered body is rejected here.)
5. **Body authority**: `apply_block_with_verdicts(ledger, era, block_cbor)`.
   Error → `Invalid { class: BodyInvalid, error: Body(e) }`, states unchanged.
   Ok((ledger', body_verdict)) → continue.
6. **Valid**: `Valid { tip: Point{slot, hash=block_header_hash}, block_no,
   body: body_verdict }`; return evolved `(ledger', chain_dep')` (chain_dep' from
   step 3's `HeaderApplied.new_state`).

> Body-hash binding (step 4) runs BEFORE body application (step 5) so an
> altered body is rejected by the cheap hash check first (pre-flight before the
> expensive tx/Plutus validation). Both are header-after gated.

## 11. Replay, Crash, and Epoch Validation

### Tests — `crates/ade_ledger/tests/block_validity_compose.rs`
- `header_before_body_fail_fast` — a block whose header fails (e.g. vrf-keyhash
  mismatch via a tampered `PoolDistrView`) → `Invalid{HeaderInvalid}`, and the
  body path is never reached (assert by a body that would otherwise be invalid
  still yielding the HeaderInvalid class).
- `altered_body_rejected_by_hash_binding` — take a real corpus block, flip a
  body byte (keep the header) → `Invalid{BodyHashMismatch}`. **The E5 negative
  test.**
- `invalid_block_leaves_state_unchanged` — for an Invalid outcome assert
  `outcome.ledger == input ledger` and `outcome.chain_dep == input chain_dep`
  (fingerprint or structural equality).
- `valid_block_evolves_both_states` — a corpus block that fully validates →
  Valid, `chain_dep` advanced (last_slot/last_block_no), `ledger` advanced.
  (If full body validation of a mainnet Conway block is too heavy for one unit
  test, use the lightest corpus block; B1-S6 does the full corpus replay.)

### Replay impact
- `block_validity` is pure; replay-equivalent.

## 13. Failure Modes

| Failure | Verdict | class |
|---|---|---|
| Block decode failure | Invalid | BodyInvalid |
| Header invalid | Invalid | HeaderInvalid |
| Body hash ≠ header body_hash | Invalid | BodyHashMismatch |
| Body invalid | Invalid | BodyInvalid |

All Invalid outcomes leave input states unchanged (no partial mutation).

### Inherited (cluster.md)

  unwired body-hash check; fail-open guards.

### Slice-specific

- Body-hash binding MUST be a real wired check proven by a negative test — not a
  comment, not an assumption that `apply_block_with_verdicts` does it.
- No new ledger validation rules — compose only.
