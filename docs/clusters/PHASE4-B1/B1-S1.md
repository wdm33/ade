## 4. Intent

Make the consensus inputs for block validation **canonical, committed, and
deterministically reproducible** — never guessed at runtime. Specifically: the
epoch nonce (eta0), each issuing pool's registered VRF key, the set-snapshot
active stake, total active stake, and the active-slots-coefficient for the
target epoch, plus the real blocks themselves.

## 9. Design Summary

### Critical facts (verified)

- `blocks_epoch577.tar.gz` is **misnamed**: it holds **14 real Conway blocks at
  slots 163,900,639 – 163,900,768**, which are the **tail of mainnet epoch 576**
  (epoch 577 begins at slot 163,900,800). Therefore the corpus epoch is **576**,
  and the matching epoch nonce is **eta0(576)**.
- **eta0(576) = `d4d5f9dc027133f27b2551711c078e7d34575daba4aa0ee7c82e4b9dd9f55c51`**,
  extracted from `s3://…/snapshots/snapshot_163468813.tar.gz :: ./state` (epoch-576
  protocol state). Method (proven): the snapshot `state` is a UTxO-HD `utxohd-mem`
  `ExtLedgerState` CBOR; scanning it for the byte pattern `82 01 58 20` (a
  non-neutral `Nonce` = `[1, bytes32]`) yields **exactly 5** contiguous matches at
  the file tail — the `PraosState` nonces in record order:
  `[evolvingNonce, candidateNonce, epochNonce, labNonce, lastEpochBlockNonce]`.
  The 3rd is eta0. (For epoch 576: evolving==candidate, a post-boundary-reseed
  corroboration.)
- Mainnet `activeSlotsCoeff = 1/20` (constant since Shelley). Confirm against the
  Conway ledger-state pparams when extracting; do not hardcode silently — record
  it in the corpus with provenance.

### Reusable nonce extractor (RED, code)

```rust
// ade_ledger::consensus_input_extract
pub struct PraosNonces {
    pub evolving: Nonce,
    pub candidate: Nonce,
    pub epoch: Nonce,           // eta0
    pub lab: Nonce,
    pub last_epoch_block: Nonce,
}
pub enum NonceScanError { NotFiveNonces { found: usize } }

/// Scan an ExtLedgerState `state` CBOR for the 5 contiguous PraosState nonce
/// wrappers (82 01 5820 <32B>). Requires EXACTLY 5; otherwise fail-fast.
pub fn extract_praos_nonces(state_cbor: &[u8]) -> Result<PraosNonces, NonceScanError>;
```

### One-time corpus extraction (throwaway tooling, documented in corpus README)

1. Download + extract `blocks_epoch577.tar.gz` → 14 block CBORs.
2. Decode each header via `ade_codec` (`decode_*_block` → `ShelleyHeaderBody`):
   collect `issuer_vkey`, `vrf_vkey`, `slot`, `block_number`, `body_hash`,
   op-cert `(sequence_number, kes_period)`. Compute `pool_id = blake2b224(issuer_vkey)`.
3. From `s3://…/extracts/ledger_state_conway576.json` (1.8 GB JSON), extract — for
   the ~14 issuing `pool_id`s only — the registered pool VRF keyhash and the
   **set-snapshot** active stake, plus the **total** active stake and `activeSlotsCoeff`.
   Use streaming (`jq`/`ijson`); do not load 1.8 GB into memory.
4. Extract eta0(576) via `extract_praos_nonces` on snapshot_163468813 `state`.
5. Write the small committed corpus (below). Delete the multi-GB downloads.

> If the `ledger_state_conway576.json` set-snapshot vs. the right epoch generation
> is ambiguous, the proof obligation is: the snapshot used for epoch-576 leadership
> is the one frozen at the epoch-574 boundary (set = mark from 2 epochs prior).
> Confirm the chosen generation reproduces a self-consistent total. Document the
> jq path used in the corpus README.

  - `extract_nonces_requires_exactly_five` — a buffer with ≠5 patterns → `NotFiveNonces`.
  - `extract_nonces_field_order` — on a committed tiny fixture (a hand-built CBOR

    `sum(pool.active_stake) ≤ total_active_stake`, all 14 block issuers resolve to
    a pool present in `pools`, and eta0 matches the pinned value.
