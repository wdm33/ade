# Slice A3 — Real forge handler in produce_mode

**Cluster:** PHASE4-N-R-A.
**Strengthens:** `CN-FORGE-01` (RequestForge contract); `CN-PROD-02`
under real load; `DC-CONS-18` body-hash binding through real forge
end-to-end; `T-KEY-01` RED side.
**Introduces:** `produce_mode::run_real_forge` + `ForgeRequestContext`
+ `SyntheticForgeInputs` + `ProducerShell::vrf_verification_key`.
**Dependencies:** PHASE4-N-R-A A2 (HEAD `a1bade3`).

## §1 Intent

Replace the S5 stub forge handler in
`produce_mode::apply_effects_with_forge_handler` with the real
**BLUE-then-RED-then-BLUE** composition pipeline. The handler now
consumes a `CoordinatorEffect::RequestForge` and produces exactly
one of `CoordinatorEvent::ForgeSucceeded | ForgeNotLeader |
ForgeFailed` — per the closed CN-FORGE-01 contract.

## §2 Pipeline implemented

```rust
pub fn run_real_forge(
    slot: u64,
    kes_period: u32,
    ctx: &ForgeRequestContext<'_>,
    shell: &mut ProducerShell,
) -> CoordinatorEvent;
```

1. **RED step 1 — VRF prove.** `shell.vrf_prove(vrf_input(slot,
   eta0, LeaderEligibility))` returns `(VrfProof, VrfOutput)`.
   Failure → `ForgeFailed { Other }`.
2. **BLUE step 2 — leader-check evaluator.**
   `verify_and_evaluate_leader(slot, eta0, vrf_vk, vrf_proof,
   leader_schedule_answer)` returns `LeaderCheckVerdict`.
   `NotEligible` → `ForgeNotLeader { slot,
   vrf_output_fingerprint }`. `Eligible` → continue. Coherence
   errors → `ForgeFailed { Other }`.
3. **RED step 3 — KES sign.** `shell.kes_sign_at(kes_period,
   &expected_vrf_input)` returns `KesSignature`. Period mismatch
   → `ForgeFailed { KesPeriodMismatch }`. **A3 honest scope:**
   the signing payload is `expected_vrf_input` (placeholder).
   The real Praos KES-signs-unsigned-header recipe is a future
   cluster's work; for A3 the placeholder is sufficient because
   the synthetic block fails `self_accept` for other structural
   reasons anyway, exercising the `ForgeFailed { SelfAcceptRejected }`
   path.
4. **GREEN step 4 — `assemble_tick`.** Stitches the signed
   artifacts into a canonical `ProducerTick`.
5. **BLUE step 5 — `forge_block`.** Constructs the block from
   the tick. Errors mapped to `ForgeFailed` via `map_forge_error`.
6. **BLUE step 6 — `self_accept`.** Full header + body
   validation against ledger + chain-dep + era schedule + pool
   view. `Accepted` → `ForgeSucceeded`. Rejected →
   `ForgeFailed { SelfAcceptRejected }`. **This gate enforces
   N1: no `ForgeSucceeded` without `self_accept = Accepted`.**

## §3 Wiring

`apply_effects_with_forge_handler` now takes a
`&SyntheticForgeInputs` parameter; the `RequestForge` arm:

1. Builds a per-slot `LeaderScheduleAnswer` whose
   `expected_vrf_input` matches `vrf_input(slot, eta0, LEADER)`
   (so `verify_and_evaluate_leader`'s coherence check passes).
2. Borrows `shell.vrf_verification_key()` so `verify_vrf` accepts
   the proof `shell.vrf_prove` emits.
3. Calls `run_real_forge`.
4. Feeds the returned `CoordinatorEvent` back into
   `coordinator_step`.

The `BroadcastBlock` arm remains a no-op — N-R-B B2 wires
`ServedChainSnapshot::push_atomic` here.

## §4 Synthetic context (A3 honest scope)

`build_synthetic_forge_context` constructs a `SyntheticForgeInputs`
that carries:

- `eta0 = Nonce(Hash32([0u8; 32]))` (placeholder).
- `leader_schedule_answer.stake_fraction = (0, 1)` — **zero
  stake**, so every slot resolves to `NotEligible` at step 2.
  This preserves the smoke test's expected 3
  `LeaderCheckOutcome` events with `is_leader: false`.
- `base_state = LedgerState::new(Conway)` (empty ledger).
- `era_schedule` with one Conway era from slot 0.
- `pool_distr_view` empty.
- Default `pparams`, `prev_opcert_counter = None`,
  `block_number = 1`, `prev_hash = [0; 32]`,
  `protocol_version = (9, 0)`.

**Promoting these to real values is N-R-B + N-R-C work:**

- N-R-B B2/B3 — push real ledger snapshots through the main loop
  (the `ServedChainSnapshot` push side feeds the BLUE `base_state`
  on the read side too).
- N-R-C C1/C2 — opcert + Conway genesis parsers feed real
  `eta0`, `kes_anchor_slot`, etc. into the loop.
- A4 — integration tests against synthetic-but-eligible
  `(stake_dist, eta0, vrf_sk)` corpora exercise the full
  Succeeded / Failed paths.

## §5 Entry gate (DQ-A2)

The cluster doc's DQ-A2 entry gate is a unit test in
`ade_ledger::producer::forge::tests::forge_block_accepts_empty_mempool`
(added in this slice). It asserts:

- `tick.mempool_tx_bytes.len() == 0`.
- `tick.mempool.accepted().len() == 0`.
- `forge_block(&tick)` returns `Ok`.
- `forged.block.tx_count == 0`.
- Exactly one `ForgeEffects::ReadyForSelfAccept` emitted with
  `next_prev_opcert_counter = 7`.

**Discipline:** if this test fails, A3 halts and the
`ProducerTick` contract is revised — A3 does NOT patch around
the failure in `produce_mode`. The test passes.

## §6 New surface additions

- `ade_runtime::producer::producer_shell::ProducerShell::vrf_verification_key`
  — derives the VK from the libsodium VRF expanded SK (bytes
  32..64). Non-secret; returned by value.
- `ade_runtime::producer::signing::VrfSigningKey::verification_key`
  — the underlying derivation.

These additions are non-breaking and preserve N9
(no secret material exposed; only the public VK).
