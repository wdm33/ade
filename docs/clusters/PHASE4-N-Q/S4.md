# Slice S4 — RED `n2n_listener` (server-role tokio wiring + handshake)

**Cluster:** PHASE4-N-Q
**Strengthens:** `CN-PROD-01` (handshake closure, server-role
direction), I1 (listener RED), I6 (closed connection lifecycle).
**Introduces:** `crates/ade_runtime/src/network/n2n_listener.rs`
(RED).
**Dependencies:** PHASE4-N-Q S3 (HEAD `d267693`); existing
PHASE4-N-G `n2n_server` reducers; PHASE4-N-L `n2n_dialer`
(precedent mirror).

## §1 Intent

Add the **server-role** TCP listener that accepts inbound peer
connections, completes the N2N handshake (`run_n2n_handshake_responder`),
and spawns a `MuxPump` per peer with `peer_role: DownstreamServer`.
The existing N-G `MuxPump` already routes
`PeerN2nServerChainSyncFrame` / `PeerN2nServerBlockFetchFrame`
events for the DownstreamServer role; S4 just plugs the listener
into that pipeline.

This is the mirror of `n2n_dialer` (PHASE4-N-L) — same
composition, opposite handshake role. The listener and the
dialer share `MuxPump`, `BlockingTransport` (handshake-window
bridge), and the `spawn_duplex` machinery.

S4 ships the listener-side tokio wiring + the handshake gate.
The actual per-peer dispatch of chain-sync / block-fetch frames
to `n2n_server::dispatch_*` is left for S5's main-loop
composition (where the served snapshot is plumbed in).

## §2 Scope

### `crates/ade_runtime/src/network/n2n_listener.rs` (RED, new)

Mirror of `n2n_dialer.rs`:

```rust
pub struct N2nListenerConfig {
    pub bind_addr: SocketAddr,
    pub our_supported: &'static [(u16, VersionData)],
    pub peer_id_generator: Arc<PeerIdGenerator>,
    pub events_out: mpsc::Sender<OrchestratorEvent>,
}

pub async fn run_n2n_listener(
    cfg: N2nListenerConfig,
    shutdown_rx: watch::Receiver<bool>,
) -> Result<(), ListenerError>;
```

Per-accepted-connection task (`run_per_peer_session`):
1. `spawn_duplex` over the `TcpStream`.
2. `tokio::task::spawn_blocking` running
   `run_n2n_handshake_responder` against a `BlockingTransport`
   bridge.
3. On handshake success: emit
   `OrchestratorEvent::PeerConnected { role: DownstreamServer, ... }`.
4. Spawn `MuxPump` with `peer_role: DownstreamServer` — pump
   emits `PeerN2nServerChainSyncFrame` /
   `PeerN2nServerBlockFetchFrame` events for downstream
   consumption.

Closed error surfaces:
- `ListenerError` — `Bind(io::ErrorKind)`, `AcceptFatal(io::ErrorKind)`,
  `Shutdown`.
- `PeerSessionError` — `Io(io::ErrorKind)`, `Handshake(HandshakeError)`,
  `Mux(MuxError)`, `OrchestratorDropped`.

API differences vs `n2n_dialer` (worth noting for future
maintenance):
- Initiator/dialer takes `VersionTable`; responder/listener
  takes `&'static [(u16, VersionData)]`. The asymmetry reflects
  the protocol — initiator proposes a list with opaque params;
  responder needs full `VersionData` to compare. Production uses
  `ade_network::handshake::version_table::N2N_SUPPORTED`.
- `MuxError` lives in `ade_network::mux::frame`, not
  `mux::transport` (small import-path quirk).

### Loopback integration test

`n2n_listener_loopback_handshake_succeeds`:
1. Bind a `TcpListener` at `127.0.0.1:0` to discover an
   ephemeral port; drop it to free the port.
2. Spawn `run_n2n_listener` against the same address.
3. Drive an in-process `N2nDialer` against the listener.
4. Collect events from both sides: must see one
   `PeerConnected { role: DownstreamServer }` (from the
   listener) and one `PeerConnected { role: UpstreamClient }`
   (from the dialer).
5. Shutdown.

The synthetic-dialer test proves the **handshake gate** closes
end-to-end. The full `dialer-fetches-forged-block` integration
is deferred to S5 (where the served snapshot is plumbed in).
The bounty-facing operator runbook is S6.

### Out of scope (deferred to S5)

- Per-peer frame dispatch (calling `n2n_server::dispatch_*`
  on `PeerN2nServerChainSyncFrame` events) — needs the served
  snapshot.
- `CoordinatorEvent::PeerRequest` event surface — needs the
  coordinator to know per-peer state, which is folded into the
  main loop in S5.
- The `dialer-fetches-forged-block` integration test — S5

## §4 Hard prohibitions

- The listener owns the socket; no other code touches it.
- Pre-handshake bytes never reach `n2n_server` reducers (the
  `BlockingTransport` is consumed by the handshake driver
  before MuxPump is constructed; the spawn_blocking task
  returns the post-handshake transport halves only).
- `n2n_listener.rs` MUST NOT import `producer::signing`,
  `KesSecret`, etc. (carries N9; same as `n2n_dialer`'s
  no-signing-dep discipline enforced by N-G's
  `ci_check_n2n_server_no_signing_dep.sh`).

## §5 Replay obligations preserved

S4 introduces no new replay obligations. The handshake
state-machine (`run_n2n_handshake_responder`) is BLUE/GREEN and
deterministic over the same `(our_supported, peer_proposal)`
inputs (existing DC-SESS-* invariants from N-L).
