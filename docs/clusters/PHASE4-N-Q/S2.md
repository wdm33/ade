# Slice S2 — GREEN coordinator + producer_log + replay test

**Cluster:** PHASE4-N-Q
**Strengthens:** `CN-PROD-02` (no secret material in coordinator
state — type-system + CI-grep enforced), `DC-PROD-02`
(coordinator slot-tick + forge-result stream replay
equivalence), `DC-PROD-01` (closed `ProducerLogEvent`
vocabulary).
**Introduces:** GREEN `coordinator` + GREEN `producer_log`
modules; new CI guard
`ci/ci_check_producer_coordinator_no_secrets.sh`.
**Dependencies:** PHASE4-N-Q S1 (planning artifacts; HEAD
`1fc21ff`).

## §1 Intent

Land the GREEN pure-state-machine coordinator that orchestrates
the slot loop + forge requests + peer lifecycle, and the GREEN
closed-vocabulary `ProducerLogEvent` enum it emits. Establish
replay equivalence (DC-PROD-02) by mechanical test.

S2 does NOT wire to tokio or to actual signing — those land in
S3 (RED producer shell) and S5 (RED main loop). S2 also does
NOT drive per-peer N2N protocol — that's S4. Lifecycle tracking
only.

After S2, the coordinator state machine is fully testable in
isolation: feed it event sequences, observe deterministic
effects + log events.

## §2 Scope

### New BLUE/GREEN files

#### `crates/ade_runtime/src/producer/coordinator.rs` (GREEN)

Pure-state-machine surface:

```rust
pub struct CoordinatorState { ... }   // no signing-key fields
pub struct CoordinatorConfig { ... }
pub struct GenesisAnchor { ... }
pub struct OpCertPublicMetadata { ... }
pub struct LedgerSnapshotRef(pub u64);
pub struct ChainTip { ... }
pub struct ForgedBlockArtifact { slot, hash, bytes }
pub struct PeerLifecycle { ... }

pub enum CoordinatorEvent { SlotTick, ForgeSucceeded,
    ForgeNotLeader, ForgeFailed, PeerConnected,
    PeerDisconnected, LedgerSnapshotUpdated,
    BroadcastDrained, Shutdown }
pub enum CoordinatorEffect { RequestForge, BroadcastBlock,
    LogEvidence }
pub enum CoordinatorError { SlotDrift, BroadcastFull,
    UnknownPeer, KesPeriodOutOfRange, UnexpectedForgeResult,
    PeerLimitExceeded, ShutdownInProgress }

pub fn coordinator_init(cfg) -> (state, Vec<Effect>);
pub fn coordinator_step(state, event)
    -> Result<(state', Vec<Effect>), CoordinatorError>;
```

**Key invariants** (mechanically enforced by §3 MACs):
- `CoordinatorState` carries no `KesSecret` / `VrfSigningKey`
  / `ColdSigningKey` field.
- `CoordinatorEvent::ForgeSucceeded` carries a
  `ForgedBlockArtifact` (slot + hash + bytes), NOT
  `AcceptedBlock`. The RED shell projects
  `AcceptedBlock → ForgedBlockArtifact` at the boundary.
- Backwards `SlotTick` is `CoordinatorError::SlotDrift`.
- `ForgeSucceeded` for a slot with no pending forge or for a
  stale slot is `CoordinatorError::UnexpectedForgeResult` (or
  `SlotMissed` evidence + dropped, when `last_slot_tick > slot`).
- Broadcast queue overflow is `CoordinatorError::BroadcastFull`
  (fail-closed per OQ3).
- Peer limit exceeded is `CoordinatorError::PeerLimitExceeded`.
- Coordinator in shutdown: all events except `Shutdown` (which
  is idempotent) return `CoordinatorError::ShutdownInProgress`.

#### `crates/ade_runtime/src/producer/producer_log.rs` (GREEN)

Closed `ProducerLogEvent` enum + closed reason enums
(`SlotMissedReason`, `PeerDisconnectReason`, `ShutdownReason`,
`ForgeFailureReason`) + opaque `PeerId(u64)`.

Per DC-PROD-01 (declared in S1):
- No `String` fields.
- No key material.
- No file paths.
- No socket addresses.
- All reasons are closed enums.
- `serde` `Serialize` derived for JSONL output (RED-side writer
  lands in S3 / S5).

### CI guard

`ci/ci_check_producer_coordinator_no_secrets.sh` (new):
1. coordinator.rs production-scope contains no
   `KesSecret`/`VrfSigningKey`/`ColdSigningKey`/`Sum6Kes::SigningKey`
   / etc. identifiers.
2. No `use producer::signing` in production scope.
3. No `use cardano_crypto::kes` or `use ade_crypto::kes_sum` in
   production scope.

Comment lines (`//`, `///`, `//!`) are excluded so doc
comments describing the prohibition don't fire the guard.

Add `serde_json` to `ade_runtime`'s dependencies (used by the

`serde_json` is already in `ade_runtime/Cargo.toml` (line 22).
No `Cargo.toml` change needed.

## §4 Hard prohibitions

- **N9.** No secret-key material in `CoordinatorState` or
  any `CoordinatorEvent`/`CoordinatorEffect` field.
  Mechanically enforced by
  `ci/ci_check_producer_coordinator_no_secrets.sh` (introduced
  in this slice).
- **N15.** `PeerId` is opaque `u64`; `ProducerLogEvent` carries
  no socket addresses.
- **N16.** `DC-PROD-02` replay claim is over canonical event
  streams only; the replay test does NOT touch wall-clock.

## §5 Replay obligations preserved + strengthened

- **DC-PROD-02** mechanically proven by
  `replay_byte_identity_across_two_runs` over a 10-event
  fixed sequence (Peer connect → 3 slot ticks with
  different outcomes → broadcast drain → peer disconnect →
  shutdown). Effects + log-event JSON byte-identical across
  runs.
