# Slice S5 — `ade_node Mode::Produce` + end-to-end composition

**Cluster:** PHASE4-N-Q
**Strengthens:** all N-Q invariants (this is the composition slice).
**Introduces:** `crates/ade_node/src/produce_mode.rs` (RED);
`Mode::Produce` + new flags in `crates/ade_node/src/cli.rs`;
dispatch in `crates/ade_node/src/main.rs`.
**Dependencies:** PHASE4-N-Q S4 (HEAD `cf01742`).

## §1 Intent

Ship the binary surface that composes the four pieces shipped in
S2..S4 (`coordinator`, `producer_log`, `producer_shell`,
`n2n_listener`) into a runnable `ade_node --mode produce`. After
S5 lands the user can:

```
ade_node --mode produce \
  --listen 127.0.0.1:3001 \
  --cold-skey cold.skey \
  --kes-skey kes.ade.skey \
  --vrf-skey vrf.skey \
  --opcert opcert.json \
  --genesis-file conway-genesis-min.json \
  --evidence-log evidence.jsonl \
  [--max-slots 100]   # smoke-test bound; omit for production
```

…and the binary will:

1. Load keys (cardano-cli envelope OR Ade-native KES envelope).
2. Load opcert + minimal Conway genesis subset.
3. Initialize the GREEN coordinator (no secret material).
4. Initialize the RED producer shell (holds keys).
5. Bind a TCP listener on `--listen`; spawn per-peer pump.
6. Run a slot-driven main loop:
   - tick the coordinator with `CoordinatorEvent::SlotTick { slot }`,
   - handle `CoordinatorEffect::RequestForge` via a stub that
     returns `ForgeNotLeader` (S5 honest scope — real forge
     against live ledger is S6 operator-runbook work),
   - handle `OrchestratorEvent::PeerConnected/Disconnected` from
     the listener,
   - write every `CoordinatorEffect::LogEvidence` line to the
     JSONL evidence log.
7. Exit cleanly on SIGINT/SIGTERM (closed reason
   `ShutdownReason::SignalReceived`) or after `--max-slots`
   ticks (`ShutdownReason::ScheduleEnded`).

## §2 Scope

### `crates/ade_node/src/produce_mode.rs` (RED, new)

- `pub fn run_produce_mode(cli: ProduceCli, shutdown_rx) -> ExitCode`.
- Key loaders: `load_kes_skey_any_format` tries cardano-cli
  envelope first, falls back to Ade-native on
  `UnexpectedType` or `MalformedEnvelope` (Ade envelope is
  binary CBOR, not JSON, so the cardano-cli loader's JSON
  parse fails before type-check).
- `parse_simple_opcert_json` and `parse_simple_genesis_json` —
  S5 honest scope. The operator runbook (S6) documents how to
  convert cardano-cli `node.opcert` (text-envelope cborHex) to
  the simple JSON shape; full cardano-cli envelope parsing
  lands in S6.
- Stub forge handler: emits `ForgeNotLeader { slot, vrf_output_fingerprint: [0; 8] }`
  back into the coordinator. Real forge integration (composes
  `shell.vrf_prove` + `shell.kes_sign_at` + `scheduler_step` +
  `self_accept`) is S6 work and requires Conway-era ledger
  state + mempool.

### Smoke test: `produce_mode::tests::produce_mode_starts_runs_three_slots_and_exits_via_max_slots`

- Generates synthetic Ade-native KES envelope + cardano-cli
  cold/VRF envelopes + synthetic opcert + minimal Conway
  genesis JSON.
- Runs `run_produce_mode` with `max_slots = 3`.
- Asserts:
  - `ExitCode::SUCCESS`.
  - First JSONL event is `CoordinatorStarted`.
  - Exactly 3 `SlotTick` events.
  - Exactly 3 `LeaderCheckOutcome` events (stub forge handler
    emits one per slot).
  - Final event is `CoordinatorShutdown { reason: "ScheduleEnded" }`.
  - No socket addresses leak into the log (N15 check).
  - No seed bytes leak into the log (key-custody check).

## §4 Doctrine alignment

- **N9** preserved — `ProduceCli` carries file paths, not secret
  bytes; secret material flows into `ProducerShell::init` and
  never crosses into `CoordinatorState`.
- **N15** preserved — `produce_mode` *operationally* needs a
  socket address (the `--listen` flag) but the evidence log
  (`ProducerLogEvent`) never carries it; `PeerId` is opaque
  `u64`. Smoke-test assertion `!content.contains("127.0.0.1")`
  enforces this.
- **N16** preserved — slot ticker uses `tokio::time::interval`
  (RED, not deterministic) but the replayable claim is the
  *event stream* into the coordinator, not the real-time
  interval; S2's coordinator replay test is the
  replay-equivalence anchor.
- **I7 (closed effect surface)** preserved — `apply_effects` and
  `apply_effects_with_forge_handler` exhaustively match
  `CoordinatorEffect::{LogEvidence, RequestForge, BroadcastBlock}`;
  S5's `BroadcastBlock` arm is a placeholder noted as S6 work.
- **Bounded-smoke-slices doctrine** — the smoke test is bounded
  (3 slots, `max_slots` cap, synthetic keys, no peer). The
  bounty artifact (cardano-node accepts Ade-forged block) is
  explicitly S6's operator-runbook deliverable, not an S5
  claim.

## §5 Exit criteria

1. `cargo build -p ade_node` succeeds.
2. `cargo test -p ade_node` clean (all existing tests +
   the new smoke).
3. `cargo clippy -p ade_node --no-deps` shows no
   produce_mode-attributable warnings.
4. `ci/ci_check_producer_coordinator_no_secrets.sh` still
   passes (S2 gate; produce_mode imports the GREEN coordinator
   but does not push secret material into it).
5. JSONL evidence file produced by the smoke run is parseable
   one-object-per-line.
