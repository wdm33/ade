## §1 Surface

```rust
pub enum OutboundCommand {
    ChainSync { peer: PeerId, reply: ChainSyncServerReply },
    BlockFetch { peer: PeerId, reply: BlockFetchServerReply },
    ClosePeer { peer: PeerId, reason: CloseReason },
}

pub enum CloseReason { Graceful, ProtocolViolation }
```

`MuxPump::outbound_relay: Option<mpsc::Receiver<OutboundCommand>>`.
`None` preserves existing dialer behavior. `tokio::select!`
in `run()` polls inbound + outbound; `std::future::pending()`
on `None` makes the outbound arm inert for dialer mode.

## §2 Encoding path

`handle_outbound_command(cmd)`:
1. Cross-peer gate: `cmd.peer() == self.peer_id`.
2. Match variant → call existing codec
   (`encode_chain_sync_message` / `encode_block_fetch_message`).
3. `dispatch_outbound_frame(mini_protocol, payload)`:
   - `mode = self.peer_role` mapped to MuxMode.
   - `session::core::step(..., ByteChunkIn::OutboundFrame
     { mini_protocol, payload, mode, timestamp: 0 })`.
   - Route resulting `SessionEffect::SendBytes` via existing
     `route_effect` — same path internal session emits use.
