# Slice B3 — PerPeerOutbound map + listener wiring

**Cluster:** PHASE4-N-S-B.
**Strengthens:** sets up `CN-PEER-OUTBOUND-MAP-01` for flip.
**Introduces:** `PerPeerOutbound` type alias +
`new_per_peer_outbound()` constructor + listener wiring.
**Dependencies:** PHASE4-N-S-B B2 (HEAD `db442f2`).

## §1 Surface

```rust
pub type PerPeerOutbound =
    Arc<RwLock<BTreeMap<PeerId, mpsc::Sender<OutboundCommand>>>>;

pub fn new_per_peer_outbound() -> PerPeerOutbound;
```

`N2nListenerConfig` and `PerPeerSessionConfig` extended
with `peer_outbound: Option<PerPeerOutbound>`. `None`
preserves existing N-Q/N-R listener behavior.

## §2 Listener wiring

`run_per_peer_session` now:

1. Completes the N2N handshake.
2. Emits `OrchestratorEvent::PeerConnected`.
3. If `cfg.peer_outbound.is_some()`:
   - Creates `mpsc::channel::<OutboundCommand>(32)`.
   - Inserts the sender into the map under `peer_id`.
   - Passes the receiver into `MuxPump::outbound_relay`.
4. Spawns the MuxPump task.

dispatch side that actually enqueues OutboundCommands +
the loopback integration test.
