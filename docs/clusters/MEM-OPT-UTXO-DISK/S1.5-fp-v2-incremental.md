### Intent
**OQ-UD-3:** `post_fp = fingerprint(ledger).combined` is computed PER BLOCK (`runner.rs:437`), and `fingerprint_utxo` is **O(n) over the sorted UTxO** (`fingerprint.rs:130`). An on-disk backend (S2) would turn that into a per-block disk scan. S1.5 replaces the UTxO fingerprint COMPONENT with a **commutative set commitment** so `post_fp` updates in O(delta)/block. **Approach A** (incremental UTxO component) per the design fork; **B (decouple per-block/checkpoint) is REJECTED** — the WAL `post_fp` must keep meaning *"state after this block,"* not "prior hash + block hash." The ledger fingerprint is **Ade-internal** (`AgreementVerdict` compares block hashes, never the fingerprint; the peer never computes it), so a **versioned re-bootstrap is allowed**.

## 3. The Construction (NAMED — part of Ade's internal replay contract)
Not "some hash." The v2 UTxO component is a fully specified commutative set commitment:

- **Name:** the **v2 UTxO set commitment** (ECMH-style group accumulator).
- **Group:** a prime-order group with a deterministic, constant-time hash-to-group. **RECOMMENDED:** Ristretto255 (`curve25519-dalek` — prime-order, no cofactor, deterministic `from_uniform_bytes` hash-to-group; integer-only, BLUE-safe). The group/dependency choice is a **slice-entry decision**, pinned forever by golden vectors once chosen. The primitive lives in `ade_crypto` (pure crypto); `ade_ledger`'s fingerprint calls it.
- **Entry commitment (binds KEY + VALUE):**
  `entry(txin, txout) = hash_to_group("ade.utxo.fp.v2.entry" ‖ canonical_txin ‖ canonical_txout)`
  where `canonical_txin` / `canonical_txout` are the EXISTING canonical CBOR encodings (`write_tx_in` / `write_tx_out`) — the same bytes the v1 fingerprint uses, so the v2 commitment binds identical canonical entry bytes.
- **Set commitment:** `utxo_component_v2 = Σ_group entry(txin, txout)` over all LIVE UTxO entries. The component `Hash32` fed to the existing rollup is `blake2b_256("ade/fp/utxo/v2" ‖ compressed_group_element)` (domain-separated), so the 7-component rollup shape and `combined` are unchanged.
- **Updates (O(delta)):** spend input → `digest -= entry(spent_txin, spent_txout)`; produce output → `digest += entry(new_txin, new_txout)`.
- **Domain separation strings (frozen):** entry tag `"ade.utxo.fp.v2.entry"`; component header `"ade/fp/utxo/v2"`; the hash-to-group domain per the chosen primitive's spec.
- **Version tag:** `fingerprint_version = 2`; the component header carries v2; `combined = rollup(era, utxo_component_v2, cert, epoch, snapshots, pparams, governance)` (only the utxo component changes).
- **Golden vectors:** pinned test vectors for `entry(txin,txout)` and for `utxo_component_v2` over a small fixed set (deterministic + order-independent + canonical) — committed, never regenerated silently.

## 4. Versioning + re-bootstrap (HARD — a replay-format version change)
- `fingerprint_version = 2`; `anchor_version = v2`; WAL `post_fp` version = v2.
- The seed import + the bootstrap compute the **v2 anchor**; the replay corpus is **regenerated under v2**.
- Old v1 evidence stays historically valid but is **NOT byte-comparable to v2** — NO silent v1/v2 comparison; NO pretending the old anchor means the same state.
- **Old v1 stores are REJECTED or require explicit re-bootstrap** — a version mismatch fails closed; **no silent mixed-version replay**.
- WAL `post_fp` uses v2 **only after a v2 bootstrap**.
- **No Cardano peer-facing change; block-hash agreement unchanged.**

## 5. Phases (two closed steps — do NOT land incremental without the oracle)

### S1.5a — v2 full recompute (the ORACLE)
- Introduce `utxo_component_v2` as a **FULL recompute** over the current BTreeMap (`Σ_group entry` over all live entries). NO incremental updates yet.
- Wire the version machinery: `fingerprint_version = 2`, the v2 component in the rollup, the anchor/WAL version tags, the v1/v2 explicit gate (mixed-version fails closed).
- **Prove:** deterministic; canonical; **order-independent** (the group sum is commutative — prove via a permutation test that shuffled insertion order yields the identical `utxo_component_v2`); versioned (v1 ≠ v2; the tag rejects mixing). Golden vectors pinned.
- **Closes** when the v2 full recompute + the version machinery + the golden vectors are green, behavior-invariant except the (intended) fingerprint-value change, and a v2 re-bootstrap produces a stable v2 anchor.

### S1.5b — v2 incremental (proven == the oracle)
- Add **per-block delta maintenance**: maintain the running group digest; on each admitted block, `digest -= entry(each spent input)`, `digest += entry(each produced output)`; derive `post_fp` from it in O(delta).
- **Prove the equivalence:** `incremental_v2_after_block == full_recompute_v2_after_block` (the S1.5a oracle) after **EVERY** tested block (a block corpus, incl. the live admission corpus).
- **Fail-closed on bad deltas:** duplicate or invalid delta application (spend of an absent entry, double-spend within a block, produce of a colliding TxIn) fails **structured + deterministic** — never a silently wrong digest.
- WAL `post_fp` now uses the incremental v2 (after a v2 bootstrap).
- **Closes** when the equivalence holds over the corpus + the live admission run, the fail-closed deltas are tested, and the replay corpus is regenerated under v2.

- **No naive XOR / integer sum** — insecure; the commitment must be a cryptographically sound commutative group accumulator (collision-resistant under add/remove).
- **The primitive is NOT an implementation detail.** It is part of Ade's internal replay contract — named construction, domain-separation strings, canonical bytes, version tag, golden vectors. All frozen.
- **No silent v1/v2 mixing** — a version mismatch fails closed.
- **Incremental (S1.5b) MUST NOT land without the full-recompute oracle (S1.5a) + the equivalence proof.**
- **No peer-facing change**; the `AgreementVerdict` (block-hash) path is untouched.
