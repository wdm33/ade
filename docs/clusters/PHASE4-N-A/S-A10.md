## 9. Design Summary

```rust
// crates/ade_network_interop/tests/live_cardano_node_session.rs
#[test]
#[ignore] // requires Docker; opted-in via -- --ignored
fn live_cardano_node_session() {
    let node = pin_docker_cardano_node_10_6_2();
    let session = ade_network::session::open_n2n(&node.peer())?;
    session.handshake_v14()?;
    session.exchange_chain_sync_to_tip()?;
    session.exchange_block_fetch_batch(10)?;
    session.submit_tx(sample_tx())?;
    session.close()?;
    write_evidence_log(&session.transcript_hash(), "docs/clusters/PHASE4-N-A/CE-N-A-5_<date>.log");
}
```

The `#[ignore]` flag ensures the test does not run in default CI (no Docker assumed). Operator runs `cargo test -p ade_network_interop -- --ignored` to validate.

### Pinning discipline (per §7 #6)

The closure gate is "pinned cardano-node 10.6.2 via Docker." A pinned digest, not a floating tag. The Dockerfile must:
- `FROM inputoutput/cardano-node:10.6.2@sha256:<digest>`
- Pin all topology and genesis files via SHA256 in the repo
- Be reproducible: any clone of the repo can rebuild and re-run the closure gate

## 11. Replay, Crash, and Epoch Validation

The closure-gate test is not "replay-equivalent across runs" in the sense the BLUE state machine is — chain tip moves, peer set rotates. The test asserts:
- Handshake succeeds at v14 (N2N) / v20 (N2C)
- Chain-sync produces non-empty signal stream that decodes
- Block-fetch returns at least N blocks
- Tx-submission2 round-trips inventory
- Session closes cleanly (no orphaned tokio task)

The transcript hash records what was exchanged so future runs can be diffed.
