# Slice B5 — main.rs `--mode admission` dispatch + CLI flags

**Cluster:** PHASE4-N-M-B
**CEs:** CE-N-M-B-7 (CN-NODE-01 strengthening — admission dispatch)
**Dependencies:** B4
**Registry effects on merge:** CN-NODE-01.strengthened_in += "PHASE4-N-M-B"; CN-ADMIT-01 stays enforced (declared in B4).

## Intent

Bind the admission runner to the `ade_node` binary as a closed
`--mode admission` operating mode. Mode dispatch stays a closed
sum across the binary (mirror N-L-LIVE's wire-only-mode
dispatch).

Add the admission-specific CLI flags + structured input wiring,
with no overlap into wire-only-mode flags. Operator pass (C)
will invoke the binary purely through this surface.

## Scope

- `crates/ade_node/src/main.rs` extension:
  - Closed `NodeMode` sum gains `Admission { ... }` variant
    (alongside existing wire-only/producer modes).
  - Mode-dispatch `match` stays exhaustive without
    `#[non_exhaustive]` (CN-NODE-01).
- `crates/ade_node/src/cli.rs` (new or extended):
  - `AdmissionCli`: `--json-seed PATH`,
    `--seed-point-slot N`, `--peer ADDR[:PORT]` (repeatable),
    `--wal-dir PATH`, `--snapshot-dir PATH`, `--log-jsonl PATH`,
    `--network-magic N`, `--genesis-hash HEX`.
  - All required, no defaults; missing flag → fatal CLI error
    pre-runner (exit code 2).
- `ci/ci_check_node_mode_closure.sh` updated to assert
  `Admission` variant + dispatch arm + no fallthrough.

## Exit codes (closed set on the binary)

| Code | Meaning                          | Source                |
|------|----------------------------------|-----------------------|
| 0    | clean shutdown                   | runner returned Ok    |
| 2    | CLI parse / validation fatal     | pre-runner            |
| 30   | live agreement diverged          | runner fatal exit     |
| 31   | input not found                  | runner fatal exit     |
| 32   | bootstrap fatal                  | runner fatal exit     |
| 33   | WAL append I/O fatal             | runner fatal exit     |

## §14 Hard Prohibitions

- No `#[non_exhaustive]` on `NodeMode`.
- No default mode fallback (`--mode` is required).
- No flag overlap between wire-only-mode and admission-mode
  flag sets (each mode owns its own struct).
- No mid-flight mode switch — mode is parsed once at startup.

## §15 Non-Goals

- No new mode beyond `Admission` in this slice.
- No `--mode reset-wal` / similar destructive flag (would
  violate DC-WAL-01 append-only).
