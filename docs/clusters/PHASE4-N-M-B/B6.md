# Slice B6 — hermetic loopback admission + admit-replay-equivalence tests

**Cluster:** PHASE4-N-M-B
**CEs:** CE-N-M-B-8 (DC-ADMIT-07 — admit-replay-equivalence true-tier)
**Dependencies:** B4, B5
**Registry effects on merge:** DC-ADMIT-07 → `enforced`.

## Intent

Land the headline true-tier proof for sub-cluster B:

> For every successful AdmittedBlock that produces a WAL
> append, replay from the anchor + WAL produces the same
> post-admit ledger fingerprint AND the same emitted
> `AgreementVerdict`.

This is the per-evidence-event strengthening of CN-STORE-03
(replay-equivalent recovery) recorded as the true-tier
recovery target in the doctrine memory
[[feedback-evidence-reducers-are-green-not-authority]].

The harness uses a hermetic loopback responder (the wire-only
shape from N-L-LIVE-test), NOT a real cardano-node, so the
test is reproducible in CI without docker / network.

## Scope

- `crates/ade_node/tests/admission_replay_equivalence.rs`
  (new integration test):
  - Spin up admission runner against an in-process loopback
    responder feeding a deterministic 3-block sequence.
  - Capture: WAL contents + JSONL admission log + final ledger
    fingerprint.
  - Restart the runner against the same seed + the captured
    WAL; assert WAL bytes unchanged, replay produces same final
    ledger fp, AND replay produces the same JSONL verdict
    transcript.
- `crates/ade_node/tests/admission_diverged_halt.rs`:
  - Loopback feeds a divergent peer-tip block;
    runner exits with `EXIT_LIVE_AGREEMENT_DIVERGED` (30).
- `crates/ade_node/tests/admission_input_not_found.rs`:
  - Loopback withholds the comparison input;
    runner exits with `EXIT_LIVE_INPUT_NOT_FOUND` (31).
- `ci/ci_check_admit_replay_equivalence.sh` — asserts the
  integration test file exists and the function names line up
  with the registry-quoted CE.

## §14 Hard Prohibitions

- No real-peer dependency in this slice (hermetic only).
- No mock that papers over a divergent fingerprint.
- No verdict transcript comparison that ignores variant
  payloads (the assertion must compare full closed-sum
  contents, not just kind discriminator).
- No `Lagging` accepted as a successful equivalence outcome
  for this proof (DC-ADMIT-08).
