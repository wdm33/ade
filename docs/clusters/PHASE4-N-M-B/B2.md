# Slice B2 — AdmissionLogEvent + writer + bidirectional vocabulary CI

**Cluster:** PHASE4-N-M-B
**CEs:** CE-N-M-B-3 (DC-ADMIT-04)
**Dependencies:** B1 (writer references `AgreementVerdict`)
**Registry effects on merge:** DC-ADMIT-04 → `enforced`.

## Intent

Closed JSONL event vocabulary for admission mode (8 variants).
Hand-rolled JSON writer (no serde-derive dep) mirroring the
wire-only `LiveLogEvent`/`LiveLogWriter` pattern shipped at
PHASE4-N-L-LIVE. Per
[[feedback-shell-must-not-overstate-semantic-truth]] the two
vocabularies are physically isolated — separate files; CI grep
enforces both directions.

## Scope

- `crates/ade_node/src/admission_log/event.rs` — closed
  `AdmissionLogEvent` sum, 8 variants:
  - `AdmissionStarted { peer_count, json_seed_path, wal_dir }`
  - `SnapshotImported { seed_point_slot, imported_utxo_fp_hex, utxo_entry_count }`
  - `BootstrapComplete { initial_ledger_fp_hex, chain_tip_slot }`
  - `BlockReceived { peer, slot, block_hash_hex }`
  - `BlockAdmitted { slot, block_hash_hex, post_fp_hex }`
  - `AgreementVerdict { kind, slot, our_hash_hex?, peer_hash_hex?, peer_slot?, tx_in_hex? }` (kind = discriminator string of the closed `AgreementVerdict` variant)
  - `AdmissionHalted { reason }` (closed `AdmissionHaltReason` sum: `Diverged | InputNotFound | WalAppendIo | BootstrapFatal`)
  - `AdmissionShutdown { reason }` (closed `AdmissionShutdownReason` sum: `SignalReceived | UpstreamDropped | OperatorRequested`)
- `crates/ade_node/src/admission_log/writer.rs` —
  `AdmissionLogWriter<W: Write>` with hand-rolled JSON
  serialization.
- `crates/ade_node/src/admission_log/mod.rs` — barrel.
- `ci/ci_check_admission_log_vocabulary_closed.sh` —
  bidirectional grep gate.

## Bidirectional vocabulary isolation

**Wire-only files** (under `ade_node/src/live_log/` +
`ade_node/src/wire_only.rs`) MUST NOT contain any of these
literals (admission-only):
- `"admission_started"`, `"snapshot_imported"`,
  `"bootstrap_complete"`, `"block_received"`,
  `"block_admitted"`, `"agreement_verdict"`,
  `"admission_halted"`, `"admission_shutdown"`

**Admission-mode files** (under `ade_node/src/admission_log/` +
`ade_node/src/admission/`) MUST NOT contain any of these
literals (wire-only-only):
- `"peer_dial_started"`, `"handshake_ok"`, `"peer_tip_read"`,
  `"peer_dial_failed"`, `"wire_smoke_complete"`

Shared (allowed in both): `"node_started"`, `"node_shutdown"`.

## §14 Hard Prohibitions

- No serde-derive dep on `ade_node` for this slice (hand-rolled
  JSON only, mirroring wire-only's discipline).
- No emitting wire-only-only literals in admission files
  (and vice versa).
- No `Healthy` / `Ready` / `Synced` event variant.
