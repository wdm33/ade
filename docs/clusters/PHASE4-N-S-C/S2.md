# Slice C2 — Preprod operator pass (bounty-facing)

**Cluster:** PHASE4-N-S-C.
**Strengthens:** conditionally narrows or flips
`CN-CONS-06` / `RO-LIVE-01` open_obligations.
**Closure:** OPERATOR-WITNESSED. Cannot close in CI.

## §1 Pre-conditions

C2 is gated on preprod stake being provisioned for the
operator's cold key:

1. The operator registers a stake pool on preprod with the
   cold/VRF/KES keys they control.
2. ADA is delegated to the pool (community delegation, or
   self-delegation from a preprod faucet).
3. The stake snapshot becomes active (2-epoch delay on
   preprod ≈ 10 days).
4. The operator's pool is eligible to be a slot leader.

Until these pre-conditions are met, C2 stays
`blocked_until_preprod_stake_available`.

## §2 Operator procedure

Identical to C1's procedure (`docs/clusters/PHASE4-N-S-C/S1.md`)
with these substitutions:

- Target: the local docker `cardano-node-preprod` container
  per [[reference-local-preprod-docker-cardano-node]] —
  `cardano-node-preprod` at `127.0.0.1:3001`.
- Genesis files: the preprod's shelley-genesis.json +
  conway-genesis.json (from `.cardano-node-preprod/config/`).
- Capture window: large enough to span at least one slot
  where the operator's pool is leader (with 1/N stake +
  ASC ≈ 0.05, expect ≈ N×20 slots between leader slots).

## §3 Pair manifest

Same schema as C1's, with `network = "preprod"`.

## §4 Conditional strengthenings

When C2 executes successfully + the paired manifest is
committed:

- `CN-CONS-06.open_obligation` narrows. The remaining
  surfaces (TxSubmission2 + mempool inclusion + multi-node
  bounty leg) are NOT closed by C2; they remain in
  `open_obligation` as separate items.
- `RO-LIVE-01.open_obligation` narrows similarly.
- `CN-OPERATOR-EVIDENCE-01` (introduced in N-S-C) flips
  `enforced` ↔ `enforced_with_evidence`.

If C2 cannot execute (preprod stake never provisioned),
the cluster closes with `blocked_until_preprod_stake_available`
on `CN-CONS-06` and `RO-LIVE-01`. This is a valid close
per the cluster plan §7 — N-S still SHIPS the engineering
surface; the bounty leg is a separate operator obligation.
