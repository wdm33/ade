# PHASE4-N-W S1 — TPraos producer-forge fail-closed

> Slice of cluster PHASE4-N-W. Addresses **CE-W-6** (I6 / N5). Surface
> reduction: narrow the producer to Praos-only before any Praos mechanics land.

## Invariant scope

A producer-forge request for a non-Praos era fail-closes with a structured,
closed reason — never a silent skip, never a TPraos producer proof. The Praos
(Babbage/Conway) forge path is untouched by this slice.

- Realizes the invariant-sketch transition `T-W1` (TPraos branch):
  `(non-Praos era, …) → fail-closed`.
- Closes nothing about the *Praos* VRF construction (that is S2); it only makes
  the unsupported-era boundary explicit.

## TCB color

- **RED** — `ade_node::produce_mode` (the guard in `run_real_forge`).
- **BLUE (additive)** — `ade_types::CardanoEra::is_praos()` classifier
  (Babbage | Conway), reused by S2's construction selector.
- **RED (vocabulary)** — `ade_runtime::producer::producer_log::ForgeFailureReason`
  gains one closed no-data variant.

## Changes

1. `crates/ade_types/src/era.rs` — add `CardanoEra::is_praos(self) -> bool`
   (`matches!(self, Babbage | Conway)`) + a unit test pinning the classification
   for all 8 eras. Pure, additive, BLUE.
2. `crates/ade_runtime/src/producer/producer_log.rs` — add closed variant
   `ForgeFailureReason::UnsupportedProducerEra` (no data, consistent with the
   existing closed no-`String` vocabulary; DC-PROD-01 preserved). This is the
   closed producer-facing realization of the sketch's
   `UnsupportedEra::ProducerForge`.
3. `crates/ade_node/src/produce_mode.rs` — at the top of `run_real_forge`
   (before the VRF prove), determine the era via
   `ctx.era_schedule.locate(SlotNo(slot))`. On `Err` → `ForgeFailed { Other }`
   (locate failure is not era policy). If the era is not Praos →
   `ForgeFailed { reason: UnsupportedProducerEra }`. Praos → proceed unchanged.

## Tests (CE-W-6)

- `crates/ade_types` unit test: `is_praos()` true for Babbage/Conway, false for
  Byron*/Shelley/Allegra/Mary/Alonzo.
- `crates/ade_node/tests/forge_succeeds.rs` (or sibling): build a
  `ForgeRequestContext` whose `era_schedule` locates a **Shelley** (TPraos) era
  at the slot, assert `run_real_forge` returns
  `CoordinatorEvent::ForgeFailed { reason: ForgeFailureReason::UnsupportedProducerEra }`
  — and that the guard fires *before* VRF/KES (no key use on the rejected path).

## Mergeability

After S1: Conway/Praos forge is unchanged (still `SelfAcceptRejected` at the VRF
step — `forge_to_self_accept_blocked_on_praos_vrf_construction` still passes);
TPraos forge is now explicit `UnsupportedProducerEra` instead of attempting a
wrong block. No invariant weakened. `vrf_input` / `VrfRole` untouched
(TPraos validation preserved — CE-W-7).

## Out of scope (this slice)

- Any change to the Praos VRF construction (alpha, leader value, `HeaderVrf`
  emission) — that is S2.
- The CI gate + registry binding — that is S3.
