# Slice A1.1 — Reference-script seed-import

**Cluster:** PHASE4-N-M-A1.1
**Strengthens:** CN-SEED-01 (release), DC-SEED-01 (derived).
**Unblocks:** DC-EVIDENCE-01 + RO-LIVE-05 (both
`enforced_scaffolding` → `enforced`).
**Dependencies:** PHASE4-N-M-A slice A1 (the existing JSON
seed importer this strengthens), PHASE4-N-M-C (which declared
the gate this slice closes).
**Registry effects on merge:** see §6 below.

## §1 Intent

Replace the `referenceScript` → `UnsupportedTxOutFeature`
fail-fast in `ade_runtime::seed_import` with a closed-shape
canonical decoder for the four cardano-node-supported script
variants. The imported `UTxOState` carries the same canonical
Babbage `script_ref` bytes that the on-chain Conway block would
carry, so the `UtxoFingerprint` is the full true fingerprint of
the operator's extracted UTxO at point P (not a fingerprint of
a silently-stripped subset).

## §2 Scope

### BLUE/GREEN file changes

- `crates/ade_runtime/src/seed_import/json.rs`:
  - Extend `RawUtxoEntry::reference_script` from a free
    `Option<serde_json::Value>` to a typed
    `Option<RawReferenceScript>` enum.
  - Add `RawReferenceScript { script: RawScriptEnvelope }`
    with `RawScriptEnvelope { cbor_hex: String, ty: String,
    description: Option<String> }`. `description` and the
    top-level `scriptLanguage` are accepted and ignored.

- `crates/ade_runtime/src/seed_import/importer.rs`:
  - Extend `JsonSeedError` with `BadReferenceScript { detail:
    &'static str }`. Remove the now-unused
    `UnsupportedTxOutFeature` variant? — NO, keep it for
    forward-compatibility; mark with `#[allow(dead_code)]` is
    NOT acceptable per IDD — instead leave it on the surface
    in case a future seed shape introduces a genuinely
    unsupported field. (See §5 for the alternative.)
  - Add `encode_script_ref(script: &RawReferenceScript) ->
    Result<Vec<u8>, JsonSeedError>` returning the canonical
    field-3 bytes:
    ```
    tag(24, bytes(cbor([variant_tag, raw_cborHex_bytes])))
    ```
    where `variant_tag` is 0/1/2/3 for SimpleScript /
    PlutusScriptV1 / PlutusScriptV2 / PlutusScriptV3, and
    `raw_cborHex_bytes` is the hex-decoded content of
    `cborHex`.
  - Wire `build_canonical_tx_out` to:
    - Read `reference_script` after the existing datum-option
      step.
    - Bump the TxOut map field count to 3 when present.
    - Append `key=3, encode_script_ref(...)` to the TxOut map
      after field 2.
  - Update the field-count computation: count starts at 2,
    +1 if datum-option, +1 if reference-script.

### Closed `type` mapping

| JSON `type`       | CBOR variant tag |
|-------------------|------------------|
| `SimpleScript`    | 0 |
| `PlutusScriptV1`  | 1 |
| `PlutusScriptV2`  | 2 |
| `PlutusScriptV3`  | 3 |

Anything else → `BadReferenceScript { detail: "unknown script type" }`.

### CBOR field-3 emission

For one entry with variant tag `T` and `cborHex` decoding to
`B` bytes (already-canonical CBOR — `bytes(plutus_binary)` for
Plutus variants, `array(...)` for native), the canonical field-3
bytes are:

```
inner   = array(2) | uint(T) | <B raw bytes>
outer   = tag(24)  | bytes(len(inner)) | inner
                  = 0xD8 0x18 | <bstr-header canonical width for len(inner)> | inner
```

`tag(24)` = major type 6, value 24, encoded as the two bytes
`0xD8 0x18`. The bytes wrapper uses the existing
`write_bytes_canonical` helper to pick the canonical width for
`len(inner)`.

The outer TxOut map gets `key=3` written via
`write_uint_canonical(3)` followed by the outer bytes raw.

## §3 Mechanical Acceptance Criteria

- [ ] `utxo_seed_accepts_plutus_v2_reference_script` —
  hand-crafted JSON entry with `referenceScript` of type
  `PlutusScriptV2` and the known mini-sample cborHex parses
  cleanly; resulting `UTxOState` has exactly 1 entry; the
  canonical TxOut bytes contain field-3 starting with
  `0x03 0xD8 0x18`.

- [ ] `utxo_seed_accepts_plutus_v1_reference_script` — same
  with `type = "PlutusScriptV1"`; field-3 inner array carries
  `uint(1)` (= 0x01).

- [ ] `utxo_seed_accepts_plutus_v3_reference_script` — same
  with `type = "PlutusScriptV3"`; field-3 inner array carries
  `uint(3)` (= 0x03).

- [ ] `utxo_seed_accepts_simple_script_reference_script` —
  same with `type = "SimpleScript"` and a small native-script
  cborHex; field-3 inner array carries `uint(0)`.

- [ ] `utxo_seed_reference_script_changes_fingerprint` — two
  imports of the SAME `TxIn` key, one with `referenceScript:
  null`, one with a PlutusScriptV2 referenceScript; the
  resulting `UtxoFingerprint`s differ.

- [ ] `utxo_seed_reference_script_deterministic_across_two_imports`
  — same JSON with reference-script entry → byte-identical
  `UtxoFingerprint` across two imports.

- [ ] `utxo_seed_rejects_unknown_script_type` — `type =
  "PlutusScriptV99"` → `JsonSeedError::BadReferenceScript {
  detail: "unknown script type" }`.

- [ ] `utxo_seed_rejects_non_hex_cbor_hex` — `cborHex = "not
  hex"` → `JsonSeedError::BadReferenceScript { detail: "cbor_hex
  not valid hex" }`.

- [ ] `utxo_seed_rejects_missing_cbor_hex` — JSON entry whose
  `script` object lacks `cborHex` → `BadReferenceScript {
  detail: "missing cbor_hex" }`. (Serde gives a `Json` error in
  practice because `cbor_hex` will be a required field on
  `RawScriptEnvelope`; that error is acceptable and the test
  asserts the closed sum it falls into.)

- [ ] `utxo_seed_mini_preprod_dump_imports_full_129_entries` —
  the committed 129-entry mini sample
  (`/tmp/utxo_sample_mini.json` if present; OR a synthetic
  fixture matching its shape) parses to exactly 129 entries,
  the reference-script entry is present, and the fingerprint
  is captured as evidence. If `/tmp/utxo_sample_mini.json` is
  not available in CI, the test compiles a synthetic 2-entry
  fixture covering the one ref-script entry; the full-129
  smoke remains evidence-only and is captured in CLOSURE.md.

- [ ] `ci/ci_check_seed_import_closure.sh` — still single
  pub `import_cardano_cli_json_utxo` in the workspace.

## §4 Hard Prohibitions

- No silent partial seed — any entry that fails to map MUST
  return a `JsonSeedError`.
- No re-encoding of the `cborHex` content. We include it raw.
- No new pub fn that converts JSON → UTxO; the existing
  `import_cardano_cli_json_utxo` is still the SOLE authority.
- No `HashMap` / `HashSet` / float / wall-clock / rand.
- No second decoder for `referenceScript` — the json.rs typed
  deserializer is the SOLE authority on the shape.

## §5 Non-Goals

- No Plutus semantic validation. Reference scripts are
  treated as opaque-typed bytes from the seed importer's
  perspective.
- No native-script semantic validation.
- No second `cborHex` canonicalization pass. If cardano-cli
  ever emits non-canonical Plutus envelopes, a separate slice
  will add a canonicalization step; until then we trust the
  cardano-node-emitted bytes.
- No removal of `UnsupportedTxOutFeature` — it stays as a
  closed variant in `JsonSeedError` for future genuine
  non-support cases. If/when a future review shows it is
  truly dead and unrecoverable, a separate slice can delete it.
