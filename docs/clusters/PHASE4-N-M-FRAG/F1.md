# Slice F1 — Per-mini-protocol payload reassembly in the session reducer

**Cluster:** PHASE4-N-M-FRAG
**New rules:** CN-SESS-04 (release), DC-SESS-06 (derived).
**Strengthens:** T-DET-01 (determinism), CN-SESS-01..03,
DC-SESS-01..05.
**Dependencies:** PHASE4-N-L (existing session reducer + mux
frame layer), `ade_codec::cbor::skip_item` (existing).
**Registry effects on merge:** see §6 below.

## §1 Intent

Port the per-mini-protocol payload reassembly pattern from
`capture_block_fetch.rs::read_for_protocol` into the closed
session reducer (`ade_network::session::core::drain_connected_frames`).
After this slice lands, the admission wire pump (and any other
consumer of `DeliverPeerFrame` events) sees ONE complete
mini-protocol CBOR item per `DeliverPeerFrame` event — never a
truncated fragment.

## §2 Scope

### BLUE file changes

- `crates/ade_network/src/session/state.rs`:
  - Add `pub struct ProtoBuffers { ten named Vec<u8> fields,
    one per AcceptedMiniProtocol variant }`.
  - `ProtoBuffers::new()` initializes every field to
    `Vec::new()`.
  - `ProtoBuffers::get_mut(&mut self, p: AcceptedMiniProtocol)
    -> &mut Vec<u8>` — closed match on the closed-sum enum.
  - `ConnectedState` gains a `pub proto_buffers: ProtoBuffers`
    field, initialized to `ProtoBuffers::new()`.

- `crates/ade_network/src/session/event.rs`:
  - Add closed-sum variant `SessionError::ProtocolPayloadMalformed
    { protocol: u16, detail: &'static str }`. No fmt::Display
    drift — extend the existing impl.

- `crates/ade_network/src/session/core.rs::drain_connected_frames`:
  - Inside the existing per-frame match arm for accepted
    mini-protocols, REPLACE the direct
    `effects.push(SessionEffect::DeliverPeerFrame { … })` with:
    1. Append `frame.payload` to
       `connected.proto_buffers.get_mut(proto)`.
    2. Loop attempting to drain complete CBOR items via
       `ade_codec::cbor::skip_item` against the per-protocol
       buffer.
    3. On `Ok((_start, end))`: snapshot bytes `[0..end]`, drain
       them from the buffer, push
       `DeliverPeerFrame { mini_protocol: proto, payload }`.
    4. On `Err(CodecError::UnexpectedEof { .. })`: break inner
       loop (need more bytes).
    5. On any other `CodecError`: return
       `Err(SessionError::ProtocolPayloadMalformed { protocol:
       proto.id(), detail })`.
  - Preserve the existing post-handshake-handshake check
    (rejected BEFORE buffer append).

### Test additions

- `crates/ade_network/src/session/core.rs` test module:
  - `fragmented_chain_sync_message_assembles_one_deliver` —
    one chain-sync IntersectFound CBOR item split across two
    mux frames bearing protocol id 2; assert exactly ONE
    DeliverPeerFrame emitted with the COMPLETE item bytes.
  - `fragmented_block_fetch_block_assembles_one_deliver` —
    one block-fetch `[4, body]` CBOR item where `body` is
    ~70KB; split across three mux frames bearing protocol id
    3; assert exactly ONE DeliverPeerFrame with the
    reassembled item.
  - `interleaved_chain_sync_and_block_fetch_fragments_stay_isolated`
    — alternating frames: cs-fragment, bf-fragment, cs-rest,
    bf-rest; assert two DeliverPeerFrame events emitted in
    completion order (cs completes first per setup, then bf).
  - `pipelined_two_chain_sync_messages_in_one_mux_frame_emit_two_delivers`
    — one mux frame whose payload is `cbor_item_a |
    cbor_item_b`; assert TWO DeliverPeerFrame events for
    protocol 2.
  - `malformed_cbor_at_item_boundary_returns_session_error` —
    inject a malformed-major-type byte at the start of a
    fragment; assert `SessionError::ProtocolPayloadMalformed`
    is returned.
  - `proto_buffers_isolation_across_all_ten_protocols` —
    feed one byte to each of the ten per-protocol buffers (via
    appropriate mux frames); assert every other protocol's
    buffer remains empty.
  - `truncated_then_complete_two_step_drain` — first call
    delivers a truncated CBOR item; second call delivers the
    rest; assert exactly one DeliverPeerFrame after the second
    call, none after the first.

## §4 Hard prohibitions

- No `HashMap` / `HashSet` for the per-protocol buffers — use
  a closed struct with one named `Vec<u8>` field per variant.
- No silent drop on truncated bytes. `UnexpectedEof` stays
  buffered.
- No drift in `DeliverPeerFrame` shape — same closed sum, same
  `mini_protocol` + `payload` fields.
- No coupling to clock / rand / float / OS-ordered iteration.
- No outbound fragmentation in this slice.
- No second per-protocol-buffer authority anywhere in the
  workspace — `ConnectedState::proto_buffers` is the SOLE
  accumulator.

## §5 Non-goals

- Block-fetch / chain-sync codec changes. The codecs already
  decode complete CBOR items correctly; the bug was at the
  layer above. FRAG fixes only the layer above.
- Outbound message fragmentation (we never send messages
  exceeding 65535 bytes today).
- Mini-protocol-level pipelining / re-ordering — frames stay
  in arrival order within a protocol.
- New mini-protocols. The closed registry stays at 10.
