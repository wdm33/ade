## 9. Design Summary

For each of the 14 corpus blocks (`corpus/validity/conway_epoch576/`):
1. Decode the block; for each tx index, assemble the on-wire tx
   `[body_i, witness_set_i, is_valid_i, aux_i]` from the block's parallel
   segments (preserving body bytes for `tx_id`).
2. Run `tx_validity(ledger, tx_cbor)` with a Conway `LedgerState`,
   `track_utxo=false` (same scope as B1's positive corpus — structural +
   supplied-witness sig validity + **tx-derived required-signer coverage**:
   explicit/withdrawal/cert/voter; input-cred coverage needs the UTxO and is
   proven synthetically in B2-S1).
3. Expect `Valid`. Record the per-tx verdict surface.

**Replay equivalence**: run the whole extraction+validation twice; assert the
verdict-surface vectors are byte-identical.

### Honest-outcome rule
Run it first and SEE. If all extracted txs are Valid → CE-B2-3 closes outright.
If some return Invalid ONLY due to the known CE-88/aiken Plutus-eval limitation
(a Phase2Invalid on a script tx) → document which, assert Valid for the rest +
header/witness validity for all, with the CE-88 carve-out named (per B1-S6). If
a tx is Invalid for ANY OTHER reason → STOP and report (a real disagreement /
bug; do not suppress). Replay-equivalence holds regardless.
