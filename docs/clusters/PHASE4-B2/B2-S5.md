## 9. Design Summary

```rust
// ade_ledger::mempool::admit
pub struct MempoolState {
    accepted: Vec<Hash32>,            // admitted tx_ids, in admission order
    accumulating: LedgerState,        // state after applying admitted txs
}
pub enum AdmitOutcome { Admitted { tx_id: Hash32 }, Rejected { class: TxRejectClass, error: TxValidityError } }

/// Tier-1: admit iff tx_validity(accumulating, tx) is Valid. On Valid, push the
/// tx_id and adopt the applied accumulating state. On Invalid, the mempool is
/// unchanged and the structured reason is returned. No false accept.
pub fn admit(mempool: &MempoolState, tx_cbor: &[u8]) -> (MempoolState, AdmitOutcome);
```

`admit` calls `tx_validity(&mempool.accumulating, tx_cbor)`:
- `Valid{tx_id, applied}` тЖТ new `MempoolState { accepted: +tx_id, accumulating: applied }`, `Admitted`.
- `Invalid{class, error}` тЖТ mempool **unchanged**, `Rejected`.

```rust
// ade_ledger::mempool::policy  (GREEN, Tier 5)
/// Deterministic eviction/ordering over admitted tx_ids. MUST NOT call
/// tx_validity or change any admit verdict тАФ it only reorders/evicts what is
/// already admitted (e.g. by fee or arrival order).
pub fn order(mempool: &MempoolState, /* policy params */) -> Vec<Hash32>;
```

Intra-block dependency: because `admit` re-validates against the **accumulating**
state, a second tx spending a first admitted tx's output validates correctly
(at track_utxo=true, the accumulating UTxO reflects the first tx).

## 11. Replay, Crash, and Epoch Validation
### Tests тАФ `crates/ade_ledger/tests/mempool_admission.rs` (synthetic, track_utxo=true)
- `valid_tx_admitted_and_accumulates` тАФ admit a valid tx тЖТ `Admitted`, accepted+1,
  accumulating state evolved.
- `invalid_tx_rejected_no_false_accept` тАФ a forged/missing-witness tx тЖТ `Rejected`,
  mempool **unchanged**.
- `admission_equals_tx_validity_verdict` тАФ for a set of txs (valid + invalid),
  `admit` outcome matches `tx_validity` verdict 1:1.
- `policy_does_not_change_validity` тАФ run `admit` with different policy/queue
  states; the admit verdict is identical (Tier-5 cannot affect Tier-1).
- `dependent_tx_admitted_against_accumulating_state` тАФ tx B spending tx A's
  output is admitted only after A (re-validated against accumulating state).
- Determinism: same sequence тЖТ same mempool + outcomes.
