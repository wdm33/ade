## 4. Intent

`(LedgerState, tx_cbor)`: phase-1 (structural + UTxO rules + witness closure)

## 9. Design Summary

```rust
// ade_ledger::tx_validity::verdict  (parallels block_validity::verdict)
pub enum TxValidityVerdict {
    Valid   { tx_id: Hash32, applied: LedgerState },
    Invalid { class: TxRejectClass, error: TxValidityError },
}
pub enum TxRejectClass { Phase1Invalid, WitnessInvalid, MissingRequiredSigner, Phase2Invalid, MalformedField }
pub enum TxValidityError {
    Decode(LedgerError),
    Witness(WitnessClosureError),        // from B2-S1
    Phase1(LedgerError),
    Phase2(LedgerError),
    MalformedField(FieldError),
}
impl TxValidityError { pub fn class(&self) -> TxRejectClass; }   // total

// ade_ledger::tx_validity::transition
pub struct TxValidityOutcome { pub verdict: TxValidityVerdict, pub applied: LedgerState }

pub fn tx_validity(ledger: &LedgerState, tx_cbor: &[u8]) -> TxValidityOutcome;
```

### Algorithm (fail-fast, total)
1. Decode tx (era-aware) → tx body + witness set. `tx_id = blake2b_256(preserved body bytes)` — never re-encode.
2. **Phase-1**: structural checks; required-signer closure (B2-S1
   `required_signers` + `verify_required_witnesses`); value/fee/validity-interval
   checks **where the UTxO resolves** (track_utxo / resolvable inputs). Any
   failure → `Invalid{class, error}`, state unchanged.
3. **Phase-2**: if the tx carries Plutus scripts, dispatch to
   `eval_tx_phase_two`. Failure → `Invalid{Phase2Invalid}`. (CE-88/aiken
   limitation: if eval is externally blocked, report honestly per B1-S6 — do
   not fake; B2-S2's synthetic tests use non-Plutus or trivially-true scripts.)
4. **Valid** → `Valid{tx_id, applied}` with the applied `LedgerState'`; Invalid
   → unchanged input state + structured reason.

Reuse the existing per-tx checks in `rules.rs`/`late_era_validation.rs` rather
than re-implementing; expose a shared per-tx phase-1 so the block body path and
`tx_validity` converge on one implementation.

## 11. Replay, Crash, and Epoch Validation

### Tests — `crates/ade_ledger/tests/tx_validity_compose.rs` (synthetic LedgerState, full control)
- `valid_tx_is_valid_and_applies` — synthetic tx + synthetic UTxO, all required
  witnesses present + value balanced → `Valid`, `applied` reflects the spend.
- `phase1_failure_short_circuits_phase2` — a phase-1 failure (e.g. missing
  witness) → `Invalid{...}` and phase-2 is never reached.
- `invalid_tx_leaves_state_unchanged` — Invalid → `outcome.applied == input`.
- `tx_id_uses_preserved_bytes` — `tx_id` equals `blake2b_256` of the preserved
  body bytes (not a re-encode).
- `class_mapping_is_total` — every `TxValidityError` variant maps to a class.
- Determinism: same `(state, tx)` twice → identical verdict.
