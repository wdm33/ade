## 4. Intent

Close the gap where the Conway/late-era body path does not verify vkey
witnesses and where required-signer **coverage** (not just supplied-witness
validity) is unchecked. Make it impossible to accept a Conway tx unless every
required signer — across a **closed, era-versioned** enumeration of sources — is
covered by a valid Ed25519 witness over the preserved tx body hash.

## 9. Design Summary

### Closed required-signer enumeration (the DC-TXV-05 surface)

```rust
// ade_ledger::tx_validity::required_signers
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum SignerSource {            // CLOSED, era-versioned
    InputPaymentKey,              // resolved input's payment key-hash credential
    ExplicitRequiredSigner,       // tx field k14
    WithdrawalKey,                // reward-account key-hash credential (k5)
    CertificateKey,               // per-cert-kind key hash
    GovernanceVoter,              // voting_procedures (k19) voter key-hash credential
    CollateralPaymentKey,         // collateral input payment key-hash (k11), script txs
}
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct RequiredSigners { pub keys: BTreeSet<Hash28> /* + provenance for diagnostics */ }

/// `resolved_inputs` supplies the spent (and collateral) outputs so input/
/// collateral payment credentials can be derived. If an input cannot be
/// resolved, that is a structured fail-fast error — NEVER a silent skip.
pub fn required_signers(
    tx_body: &ConwayTxBody,
    resolved_inputs: &ResolvedInputs,   // from LedgerState.utxo_state
    era: CardanoEra,
) -> Result<RequiredSigners, RequiredSignerError>;
```

> Script-hash credentials (inputs/collateral whose address credential is a
> script) are NOT vkey-required-signers — they are phase-2 / native-script
> obligations (out of this slice). Only key-hash credentials are required vkey
> signers. Stake/DRep/committee/pool cert key-hashes per the Conway cert kinds:
> pin each against the spec.

### Fail-closed coverage

```rust
// ade_ledger::tx_validity::witness
/// Every required key hash must be covered by a witness whose Ed25519 sig over
/// the PRESERVED tx body hash verifies. Wrong-size key/sig -> Err (from_bytes,
/// fail-closed). An extra irrelevant witness never substitutes for a missing one.
pub fn verify_required_witnesses(
    body_hash: &Hash32,
    required: &RequiredSigners,
    witnesses: &[VKeyWitness],
) -> Result<(), WitnessClosureError>;
```
`WitnessClosureError` (closed): `MissingRequiredSigner { key_hash, source }`,
`InvalidWitnessSignature { key_hash }`, `MalformedWitnessField { which, .. }`,
`UnresolvableInput { .. }`.

### Body-path wiring (gap fix)

Wire `required_signers` + `verify_required_witnesses` into the Conway/late-era
body validation in `rules.rs`/`late_era_validation.rs`. **track_utxo note**:
input/collateral payment-key coverage needs the resolved UTxO. When the body
path runs with `track_utxo=true` it performs full coverage; the tx-derived
sources (explicit, withdrawal, certificate, governance voter) are checked
**regardless** of `track_utxo` (they need no UTxO). The B1 positive corpus runs
`track_utxo=false` — so its 14 blocks must still pass (only tx-derived coverage
+ supplied-witness sig validity apply there; do NOT regress them). The full
input-cred coverage is proven synthetically here and exercised on real UTxO in
B2-S3.

## 11. Replay, Crash, and Epoch Validation

### Tests — `crates/ade_ledger/tests/tx_witness_closure.rs` (synthetic fixtures, full control)
Construct a synthetic Conway tx + a synthetic mini-UTxO (controlled input
addresses → known required key hashes) + valid Ed25519 key material. **Hard gates**:
- `all_required_covered_is_valid` — every source covered by valid witnesses → Ok.
- `missing_input_payment_witness_rejected` → `MissingRequiredSigner{InputPaymentKey}`.
- `missing_explicit_required_signer_rejected` → `{ExplicitRequiredSigner}`.
- `missing_withdrawal_witness_rejected` → `{WithdrawalKey}`.
- `missing_certificate_witness_rejected` → `{CertificateKey}`.
- `missing_governance_voter_witness_rejected` → `{GovernanceVoter}`.
- `wrong_size_signature_rejected` → `MalformedWitnessField` / fail-closed.
- `signature_over_wrong_body_rejected` → `InvalidWitnessSignature`.
- `witness_correct_key_wrong_body_rejected` → `InvalidWitnessSignature`.
- `extra_irrelevant_witness_does_not_substitute` — an unrelated valid witness
  present but a required one missing → still `MissingRequiredSigner`.
- `unresolvable_input_is_fail_fast` → `UnresolvableInput` (never silent skip).
- `script_credential_input_not_a_vkey_signer` — script-hash input is not added
  to required vkey signers.
- Determinism: same inputs twice → same result.

### Body-path regression
- A test (or reuse B1's positive corpus) confirms the 14-block Conway corpus
  still validates with the wiring in place (track_utxo=false path).

## 14. Hard Prohibitions

### Inherited (cluster.md)
- Skipping witness verification; omitting a signer source; verifying supplied
  witnesses without coverage; fail-open guards; re-encode for hashing;
  `HashMap`/float in BLUE; softening to pass a tx.

### Slice-specific
- `SignerSource` is a closed enum — a new source requires an explicit versioned addition.
- An unresolvable input is fail-fast, never a silent "assume covered."
- Script-credential inputs are NOT vkey signers (don't over-require → false reject).
