# Slice A3 — WAL closed sum + WalStore trait + FileWalStore

**Cluster:** PHASE4-N-M-A
**CEs:** CE-N-M-A-4 (CN-WAL-01 + DC-WAL-01), CE-N-M-A-5 (DC-WAL-02)
**Dependencies:** A2 (BootstrapAnchor for verify_chain init)
**Registry effects on merge:** CN-WAL-01 + DC-WAL-01 + DC-WAL-02 → `enforced`.

## Intent

Append-only WAL that records every forward-step (admit-block) as
a closed-sum entry with a fingerprint chain. Verify-chain walks
the log and asserts integrity. No truncate / rewrite / replace
on the trait or impls (append-only **by type**).

## Scope

- **BLUE** `crates/ade_ledger/src/wal/event.rs`:
  - `pub enum WalEntry` closed sum. Single variant in this
    cluster: `AdmitBlock { prior_fp, block_hash, slot, verdict, post_fp }`.
  - `pub enum BlockVerdictTag` (`Valid` | `Invalid` — mirrors
    the existing `BlockValidityVerdict` discriminant; carries no
    payload for compact serialization).
- **BLUE** `crates/ade_ledger/src/wal/store_trait.rs`:
  - `pub trait WalStore { fn append(...); fn read_all(...);
    fn verify_chain(anchor) -> Result<(), WalError>; }`.
  - **No** `truncate` / `rewrite` / `replace` / `delete`
    methods.
- **BLUE** `crates/ade_ledger/src/wal/replay.rs`:
  - `pub fn replay_from_anchor(...)` — pure replay reducer
    composing `WalEntry`s into a `LedgerState`.
- **BLUE** `crates/ade_ledger/src/wal/error.rs`:
  - Closed `WalError` sum (`Io`, `Decode`, `ChainBreak`,
    `MissingPriorEntry`, `BlockBytesMissing`).
- **GREEN** `crates/ade_runtime/src/wal/file_wal_store.rs`:
  - `pub struct FileWalStore` impl `WalStore`.
  - On-disk: one file per cluster of entries; rotation at 10K
    entries or 100 MB; CRC32C 4-byte suffix per file.
  - `fdatasync` after every `append`.

## §12 Mechanical Acceptance Criteria

- [ ] `wal_entry_admit_block_round_trips_canonical_cbor`.
- [ ] `wal_store_trait_has_no_mutation_other_than_append` —
  introspection at compile time; the trait's pub items are
  enumerated.
- [ ] `wal_verify_chain_passes_on_valid_chain` — 3 entries with
  matching `prior_fp/post_fp`; verify returns `Ok`.
- [ ] `wal_verify_chain_catches_break` — corrupt entry 2's
  `prior_fp`; verify returns `Err(ChainBreak)`.
- [ ] `wal_verify_chain_anchors_first_entry_to_initial_ledger_fp`
  — first entry's `prior_fp` must equal
  `anchor.initial_ledger_fingerprint`.
- [ ] `file_wal_store_append_then_read_all_round_trips`.
- [ ] `file_wal_store_rotates_at_10k_entries`.
- [ ] `file_wal_store_crc32c_catches_bitflip`.
- [ ] `ci/ci_check_wal_append_only.sh` —
  - `WalStore` trait has methods named `append`, `read_all`,
    `verify_chain` only. No `truncate` / `rewrite` / `replace` /
    `delete` / `clear`.
  - No `impl WalStore` adds non-trait methods with those names.

## §14 Hard Prohibitions

- No `WalStore::truncate` / `rewrite` / `replace` / `delete` /
  `clear` (trait or impl).
- No wall-clock / rand / float in any wal/*.rs file.
- No HashMap / HashSet (BTreeMap if needed).
- No second `pub trait WalStore`.
- No bypass of `verify_chain` in any caller — every
  `WalStore` consumer that reads the log must verify.

## §15 Non-Goals

- No multi-WAL-file consolidation / compaction (Tier-5 future).
- No remote / distributed WAL backends.
- No `RollBackward` / `CaptureSnapshot` variants (additive,
  future).
