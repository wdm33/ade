# Slice A5 — Honest-scope CI gate + cluster close

**Cluster:** PHASE4-N-M-A
**CEs:** CE-N-M-A-7 (RO-GENESIS-REPLAY-01 honesty)
**Dependencies:** A4.
**Registry effects on merge:**
- `RO-GENESIS-REPLAY-01` stays `declared` with
  `open_obligation = "blocked_until_genesis_replay_cluster"` —
  CI asserts the open obligation is intact (prevents
  accidental "we did genesis replay" drift in future commits).
- `RO-MITHRIL-IMPORT-01` carried unchanged.
- Existing rules strengthened with `PHASE4-N-M-A`: T-DET-01,
  CN-CONS-08.

## Intent

Prevent the cluster's honest-scope claims from drifting. The
mechanical evidence in A1-A4 says "Ade bootstraps from an oracle
seed at P and replays-equivalently from the anchor". It does NOT
say "Ade replays from genesis". A CI gate watches the registry
to make sure no future commit weakens the open obligation on
`RO-GENESIS-REPLAY-01` without doing the actual work.

## Scope

- `ci/ci_check_genesis_replay_open_obligation.sh` —
  - Greps registry for `RO-GENESIS-REPLAY-01`.
  - Asserts `status = "declared"` AND
    `open_obligation = "blocked_until_genesis_replay_cluster"`.
  - If a future commit flips it to `enforced` without the
    obligation key, this gate catches it and demands the
    backing slice doc + tests.

## §12 Mechanical Acceptance Criteria

- [ ] `ci/ci_check_genesis_replay_open_obligation.sh` passes
  with the current registry.
- [ ] Cluster close commit applies the strengthenings to
  T-DET-01 + CN-CONS-08.

## §14 Hard Prohibitions

- No claim in any cluster artifact (`cluster.md`, `CLOSURE.md`,
  commit message) that "Ade replays from genesis" or
  "Ade independently validates from byron".

## §15 Non-Goals

- This slice does NOT implement genesis replay. That's a
  separate cluster (multi-month). This slice only prevents
  drift on the open obligation.
