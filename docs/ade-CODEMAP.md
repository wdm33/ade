## Conventions

- A **module** in Ade is a Cargo workspace crate (smallest independently-buildable unit). Source: `Cargo.toml` `[workspace] members` (11 entries). One exception to crate-granularity: `ade_network` is split by *submodule color* ŌĆö `.idd-config.json` `core_paths` resolves BLUE at the submodule path level (9 paths), so its BLUE / GREEN / RED submodules are documented as separate entries.
- Several BLUE crates host sub-trees that carry a *different* TCB color by their own module banner and cluster TCB Color Map; and the RED `ade_runtime` and `ade_node` crates host numerous **GREEN-by-content** sub-trees. These are surfaced as sub-classification notes inside the owning crate's entry (or, where load-bearing, as their own GREEN/RED entries below).
- Modules are listed by TCB color (BLUE ŌåÆ GREEN ŌåÆ RED), alphabetical within each color.
