## 1. Surface Reduction Rules

> External inputs reduce to canonical form before entering authoritative pipelines. Ade's external surfaces are the N2N/N2C wire, operator files, and `argv`. Each reduces to a canonical type before any BLUE authority sees it. **The native Mithril bootstrap added a new operator-file INGRESS** (the V2 LedgerDB `state`/`tables` + manifest + Shelley genesis) — but it reduces to BLUE canonical authority through the single closed native composition; the raw cardano-node CBOR is never authority. **ECA added NO new ingress surface.** Its activation candidate is derived ENTIRELY from canonical DURABLE state already inside the node (the durable selected-chain ChainDB window + the manifest-bound bootstrap checkpoint) — there is no new external surface to reduce, and crucially **no flag/env/peer/CLI feed may supply or decide the candidate.**

**Rule:** New ingress surfaces attach by producing the canonical type's bytes and entering the same authoritative pipeline. They **may not** introduce new pipeline steps, reorder existing steps, or shortcut into the core via a back door.

## 4. Version-Gated vs. Frozen Contracts

### Frozen (immutable at current version — change = new major version)

- **Wire format / encoding**: Cardano-canonical CBOR via `minicbor` + the `ade_codec` canonical primitives — field order = wire order for hash-bearing structures; **wire bytes are preserved, never re-encoded for hashing** (`ci_check_hash_uses_wire_bytes.sh`). The native Mithril `tables` materialization embeds inline-datum / reference-script wire bytes VERBATIM (tag-24), never re-decoded.
- **Tag-24 envelope**: `0xd8 0x18` CBOR-in-CBOR — the SOLE `ade_codec::cbor::tag24::{wrap_tag24, unwrap_tag24}` authority (`CN-WIRE-08/12`). The native Mithril UTxO materialization routes its datum/script byte embedding through the matching `wrap_tag24`.
- **Hash algorithms**: `blake2b_256` / `blake2b_224` (`ade_crypto::blake2b`) — immutable per version; the single body-hash recipe `block_body_hash`. The v2 `UtxoSetCommitment` reuses `blake2b_256`/`blake2b_512`. The `EpochConsensusView::canonical_hash` and the native Mithril `fingerprint_utxo_v2`/manifest-artifact hashes all reuse `blake2b_256`.
- **The v1 UTxO fingerprint (`fingerprint_v1`) is FROZEN** — the v1 goldens + the 12 boundary-snapshot pins repointed at it (`DC-MEM-10`).
- **The UTxO canonical fingerprint is computed over canonical keys, never storage order, never the allocator** (`DC-MEM-05/06`).
- **The header VRF / KES / DSIGN verification recipes**: `ade_crypto` (Praos VRF draft-03, the Ade-owned Sum6KES, Ed25519). **VRF strength is FROZEN.** The `DC-CRYPTO-10`/`OP-OPS-04` KES-period gate (the opcert-anchored RELATIVE evolution) tightened the forge's KES sign without touching the verify recipe.
- **`select_best_chain`** (the `DC-CONS-03` fork-choice contract): **FROZEN — byte-unchanged across the span.** The single selector.
- **The `#![deny(unsafe_code)]` workspace invariant** — preserved with ZERO local allows in `ade_node`/BLUE/GREEN; the SOLE `unsafe` is quarantined in `ade_mem_diag`.
- **The durable WAL grammar is APPEND-ONLY and tag-frozen**: `WalEntry` closed sum — tag 0 `AdmitBlock`, tag 1 `RollBack`, tag 3 `SeedEpochConsensusInputsImported`, tag 4 `EpochConsensusViewActivated`; **tag 2 reserved, never reused.** Existing tags' encodings are frozen; a new tag is the only growth path.
- **The closed era enum** `CardanoEra` (Byron=0 … Conway=7); the closed `PrevHash = Genesis \| Block(Hash32)`.
- **The output asset-quantity domain** `OutputAssetQuantity(u64)` — non-negative Word64, canonical unsigned CBOR, checked add/sub only; type-incompatible with `MintBurnQuantity(i64)` (`DC-LEDGER-VALUE-01`).
- **All 508 canonical types**: existing wire formats frozen; new types may be added (the +21 this span — the leadership-complete view, the native Mithril decoders/materialization, the value/param types — are all closed-by-construction).

### Version-gated (can evolve across major versions)

- **The bootstrap manifest is version-gated by an explicit domain byte** — `0xB0` (v1, 142 bytes) → **`0xB1` (v2, 174 bytes — adds the bound UTxO-seed hash)**. v1 stays decodable (`is_v2()` discriminates); the native Mithril authority transition emits v2. A new committed field = the next domain byte + a `DC-EVIEW-09` strengthening; the seed and cert-state stay separately typed.
- **The `WalEntry` sum is version-gated by APPEND-ONLY tag** — a new durable forward-step = the next free tag + canonical encode/decode + a versioned WAL change + a replay-equivalence corpus (`ci_check_eview_activation_wal.sh` / `ci_check_wal_rollback_replay_equiv.sh`). **Never widen an existing variant; tag 2 stays reserved.**
- **The `EpochConsensusView` bound-view shape is version-gated** — it is now LEADERSHIP-COMPLETE (the VRF-keys gap is CLOSED); any further binding field = a `canonical_bytes`/golden bump + a `DC-EVIEW-07` strengthening, never a parallel VRF index.
- **The `MinUtxoRule` is version-gated by era** — a new era's min-UTxO rule is a new variant + a `DC-LEDGER-PARAMS-01` strengthening; the absolute-floor and per-byte variants stay distinct.
- **The reduced-checkpoint redb layout is version-gated** — a new table/META marker = a `DC-EPOCH-11` strengthening + the layout gate; the fixed-width key + canonical-order invariant are preserved.
- **The v1→v2 UTxO-fingerprint cutover (carried)** — an evolution of Ade's INTERNAL durable-replay contract (`DC-MEM-10`), never peer-facing. A future v3 = a new version tag + new goldens + a fail-closed old-store reject.
- New variants in the closed `AdmissionLogEvent` vocabulary (currently 23): require a `DISCRIMINATORS` allow-list entry + the closed-vocabulary CI gates. Observe-only.
- New fail-closed discriminants across the span + carried closed enums (`ActivationReject`, `EpochViewActivationError`, `EpochRebindReject`, `SourceWindowError`, `CandidateDeriveError`, `BootstrapManifestError`, `LedgerDbStateError`, `TablesDecodeError`, `TxOutMaterializeError`, `MithrilNativeAssemblyError`, `NativeMithrilBootstrapError`, `CertExtractError`, `NativeFirstRunError`, `StakeRefReject`, `WindowDriverError`, `StaticUtxoFpError`, `ShedReason`, `EvidenceDefect`, …): require the matching fail-closed CI gate + a `DC-*` strengthening.
