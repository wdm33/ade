## 1. Surface Reduction Rules

> External inputs reduce to canonical form before entering authoritative pipelines. Ade's external surfaces are the N2N/N2C wire, operator files, and `argv`. Each reduces to a canonical type before any BLUE authority sees it. **MEM-OPT-UTXO-DISK added NO new ingress surface** — it changed the runtime memory profile of the operator-file UTxO-seed reduction (PATH A: compute the constant `StaticUtxoFp` once, then `drop` the in-memory UTxO) WITHOUT changing its canonical OUTPUT, and it deepened the fail-closed posture on the store-open path (`FingerprintVersionMismatch`). The streaming reduction MECHANISM itself (whole-buffer → streaming + `DuplicateTxIn`) was MEM-OPT-OPS S2, carried.

**Rule:** New ingress surfaces attach by producing the canonical type's bytes and entering the same authoritative pipeline. They **may not** introduce new pipeline steps, reorder existing steps, or shortcut into the core via a back door.

### Surface: N2N inbound wire (received blocks/headers/txs/rollbacks — the LIVE `--mode node` feed)

### Frozen (immutable at current version — change = new major version)

- **Wire format / encoding**: Cardano-canonical CBOR via `minicbor` + the `ade_codec` canonical primitives — field order = wire order for hash-bearing structures; **wire bytes are preserved, never re-encoded for hashing** (`ci_check_hash_uses_wire_bytes.sh`). - **Hash algorithms**: `blake2b_256` / `blake2b_224` (`ade_crypto::hash`) — immutable per version; the single body-hash recipe `block_body_hash`. **The v2 `UtxoSetCommitment` reuses `blake2b_256` / `blake2b_512` over canonical `TxIn`/`TxOut` bytes** (allocator/parse-order-independent), never the allocator's allocation addresses.
- **The v1 UTxO fingerprint (`fingerprint_v1`) is FROZEN** — the v1 goldens + the 12 boundary-snapshot pins repointed at it (`DC-MEM-10`). It is retained as the FROZEN prior version of Ade's internal replay contract.
- **The UTxO canonical fingerprint is computed over canonical keys, never storage order, never the allocator** (`DC-MEM-05/06`) — the streaming and whole-buffer seed-import paths agree byte-for-byte; the dormant redb backend's fixed-width key forces byte-sorted iteration == canonical `TxIn` order.
- **The header VRF / KES / DSIGN verification recipes**: `ade_crypto` (Praos VRF draft-03, the Ade-owned Sum6KES, Ed25519). **VRF strength is FROZEN.**
- **`select_best_chain`** (the `DC-CONS-03` fork-choice contract): **FROZEN — byte-unchanged by N-AO AND both MEM-OPT clusters.** The single selector.
- **The process allocator is determinism-neutral, not a frozen contract**: the `#[global_allocator]` (mimalloc) MAY be swapped — what is FROZEN is that the allocator is INVISIBLE to BLUE (`DC-MEM-06`).
- **The `#![deny(unsafe_code)]` workspace invariant** — preserved with ZERO local allows in `ade_node` (and every BLUE/GREEN crate); the SOLE `unsafe` (mimalloc `mi_collect`) is quarantined in `ade_mem_diag`, the one crate that does NOT carry the deny (it IS the quarantine; `ci_check_mem_diag_quarantine.sh`).
